# AI5 · RAG 检索增强生成

> AI 时代专题 · 需求 → 原理 → 流程 → 前端角色 → 考点

## 需求场景

大模型有两个硬伤:① **知识截止**(不知道训练之后的事、不知道你公司的私有数据);② **幻觉**(一本正经胡编)。**RAG(Retrieval-Augmented Generation,检索增强生成)** 让模型基于"检索到的真实资料"回答,解决这两个问题——是"AI + 企业知识库/文档问答"的核心方案。

## 原理

不把知识塞进模型(那要微调,成本高),而是**在提问时把相关资料检索出来,拼进 prompt** 让模型参考回答。

```
传统:用户问题 → 模型(靠训练时的记忆,可能过时/幻觉)

RAG:用户问题 → 检索相关资料 → (问题 + 资料)一起给模型 → 基于资料回答
```

## 完整流程

### 阶段一:知识库构建(离线,预处理)

```
文档(PDF/网页/Markdown)
   ↓ 切分(Chunking):切成小段
   ↓ 向量化(Embedding):每段转成向量(语义的数字表示)
   ↓ 存入向量数据库(Pinecone / Milvus / pgvector)
```

### 阶段二:检索 + 生成(在线,每次提问)

```
用户问题
   ↓ 把问题也向量化
   ↓ 在向量库里做相似度���索(找语义最接近的 Top-K 段落)
   ↓ 把检索到的资料 + 问题 拼成 prompt
   ↓ 交给大模型生成答案(基于资料,减少幻觉)
   ↓ (可选)附上引用来源
```

### 核心概念

| 概念 | 含义 |
|------|------|
| **Embedding** | 把文本转成向量,语义相近的向量距离近 |
| **向量数据库** | 存向量、支持相似度检索 |
| **Chunking** | 把长文档切成小段(太大超上下文,太小丢语义) |
| **Top-K** | 检索最相关的 K 段 |
| **相似度** | 余弦相似度等衡量语义接近程度 |

## 前端在 RAG 里的角色

前端通常不做向量检索(在后端),但负责:
1. **知识库管理界面**:上传文档、查看切分、管理来源;
2. **对话界面**:流式展示回答(见 AI2/AI3);
3. **引用来源展示**:显示答案基于哪些文档片段(可点击溯源),增强可信度;
4. **反馈机制**:用户标记答案有用/无用,优化检索。

```jsx
// 展示带引用来源的回答
function Answer({ content, sources }) {
  return (
    <div>
      <Markdown>{content}</Markdown>
      <div className="sources">
        参考来源:
        {sources.map(s => (
          <a key={s.id} href={s.url}>{s.title}(相关度 {s.score})</a>
        ))}
      </div>
    </div>
  );
}
```

## 考点

**Q:RAG 解决什么问题?**
A:解决大模型的知识截止(不知道新信息和私有数据)和幻觉(编造)。通过在提问时检索真实资料拼进 prompt,让模型基于可靠资料回答,而不是靠训练记忆瞎编。

**Q:RAG 和微调(fine-tune)怎么选?**
A:RAG 适合知识频繁更新、需要溯源、私有文档问答——改知识库即可,无需重训、成本低、可引用来源;微调适合改变模型的行为风格/固定能力。知识类问答优先 RAG,两者也可结合。

**Q:Embedding 和向量数据库是什么?**
A:Embedding 把文本转成能表示语义的向量,语义相近的文本向量距离近;向量数据库存这些向量并支持高效的相似度检索,用来根据问题找最相关的资料段落。

**Q:前端在 RAG 系统里做什么?**
A:知识库管理(上传/管理文档)、对话界面(流式渲染)、**引用来源展示**(答案溯源、提升可信度)、用户反馈收集。检索和向量化在后端。

**Q:Chunking 为什么重要?**
A:文档切太大超出上下文窗口、检索不精准;切太小丢失上下文语义。合理的切分(按语义/段落、适当重叠)直接影响检索质量和最终答案准确性。

## 一句话总结

**RAG 解决大模型知识截止和幻觉:离线把文档切分(Chunking)→向量化(Embedding)→存向量库;在线把问题向量化→检索 Top-K 相关段落→拼进 prompt→模型基于真实资料回答并可溯源;比微调更适合知识频繁更新+私有文档问答(改库即可、成本低);前端负责知识库管理、流式对话、引用来源展示、反馈收集。**
