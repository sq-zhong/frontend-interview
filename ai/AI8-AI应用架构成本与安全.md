# AI8 · AI 应用架构、成本与安全

> AI 时代专题 · 架构 → 成本 → 安全 → 体验 → 考点

## 需求场景

把前面的点(接入、流式、工具、RAG、提示词)串成一个**生产可用的 AI 应用架构**,并回答工程上最现实的三个问题:**架构怎么搭、成本怎么控、安全怎么保**。这是 AI 系统设计题的综合考察。

## 整体架构

```
前端(对话 UI / 流式渲染 / 工具映射)
        ↓  只调自己的后端,绝不直连大模型
BFF / AI 网关(鉴权、限流、计费、审计、内容审核、路由)
        ↓
  ┌─────────────┬──────────────┬─────────────┐
大模型服务      RAG 检索        工具/函数执行
(可多模型路由)  (向量库)        (调内部系统)
```

**AI 网关**是核心中间层:统一管理密钥、限流、多模型路由(按成本/能力选模型)、缓存、审计、内容安全。

## 成本控制(务实且高频)

大模型按 token 计费,不控制成本会失控:

| 手段 | 说明 |
|------|------|
| **模型分级** | 简单任务用小/便宜模型,复杂才用大模型 |
| **上下文裁剪** | 只带必要历史;长对话做摘要压缩 |
| **缓存** | 相同/相似问题命中缓存(含语义缓存),不重复调用 |
| **限流限额** | 按用户/租户设配额,防滥用和盗刷 |
| **流式提前停止** | 用户满意可中断生成,省 output token |
| **max_tokens** | 限制输出长度 |
| **请求去重防抖** | 避免重复提交(见 W3) |

## 安全(多层防护)

**1. 密钥安全**:API Key 只在服务端,前端绝不接触(见 AI1);
**2. 鉴权限流**:AI 接口要登录鉴权 + 限流,防盗刷(一次盗刷可能烧掉大量费用);
**3. 输入安全**:防 Prompt 注入(见 AI7)、过滤敏感词、限制输入长度;
**4. 输出安全**:AI 输出当不可信数据——渲染防 XSS(见 AI3/N6)、内容审核(涉黄涉政过滤)、工具调用做权限校验和二次确认(见 AI4);
**5. 数据隐私**:用户数据/公司机密是否会被送到第三方模型?脱敏、私有化部署、合规(见下)。

## 体验优化

- **流式输出**:降低感知延迟(见 AI2),首字时间(TTFT)是关键指标;
- **加载态**:思考中动画、骨架屏;
- **停止/重新生成**:给用户控制权;
- **错误兜底**:模型超时/限流时降级(排队提示、切备用模型、缓存回答);
- **反馈机制**:点赞点踩收集数据优化。

## 考点

**Q:一个生产级 AI 应用的架构怎么设计?**
A:前端只做对话 UI 和流式渲染,调自己的后端;中间是 **AI 网关/BFF**,负责鉴权、限流、计费、审计、内容审核、多模型路由和缓存;后面接大模型、RAG 检索、工具执行。核心是不让前端直连大模型,所有管控集中在网关层。

**Q:AI 应用怎么控制成本?**
A:模型分级(简单任务用便宜模型)、上下文裁剪和摘要、语义缓存复用相似问答、按用户限流限额、支持流式提前停止、限制 max_tokens、请求去重。因为按 token 计费,不控会失控甚至被盗刷。

**Q:AI 应用有哪些安全风险?**
A:密钥泄露(前端不能有 key)、接口被盗刷(要鉴权限流)、Prompt 注入、输出 XSS、工具调用越权、数据隐私泄露(敏感数据送第三方模型)。要多层防护:输入过滤、输出净化+审核、工具权限校验、数据脱敏、后端二次鉴权。

**Q:AI 功能的体验关键指标?**
A:首字时间(TTFT,靠流式降低感知延迟)、流式输出、明确的加载/思考态、可停止可重试、失败降级(备用模型/缓存)、反馈机制。

**Q:用户隐私数据怎么处理?**
A:评估数据是否能送第三方模型;敏感字段脱敏后再送;高敏感场景用私有化部署或专有模型;遵守合规要求;明确告知用户数据用途。

## 一句话总结

**生产级 AI 架构:前端(对话/流式)→ AI 网关/BFF(鉴权/限流/计费/审计/内容审核/多模型路由/缓存)→ 大模型+RAG+工具,前端绝不直连大模型;成本靠模型分级+上下文裁剪+语义缓存+限额+提前停止;安全多层:密钥服务端、接口鉴权限流防盗刷、防注入、输出净化审核、工具越权校验、数据脱敏;体验靠流式(TTFT)、可停止、失败降级。**
