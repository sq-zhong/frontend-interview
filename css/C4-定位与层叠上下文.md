# C4 · 定位与层叠上下文

> 热度:🔥🔥

## 一句话考点

`position` 有 5 种值(static/relative/absolute/fixed/sticky);`z-index` 只在**层叠上下文**内比较,元素能否遮盖取决于是否形成新的层叠上下文,而非单纯 z-index 大小。

## 原理精讲

### 1. position 五种值

| 值 | 定位基准 | 是否脱离文档流 |
|----|---------|--------------|
| `static` | 默认,正常流 | 否 |
| `relative` | 相对**自身**原位置偏移 | 否(占位保留) |
| `absolute` | 相对最近的**非 static 祖先** | 是 |
| `fixed` | 相对**视口** | 是 |
| `sticky` | 滚动到阈值前 relative,之后 fixed | 否 |

### 2. 层叠上下文(Stacking Context)

一个独立的三维空间,内部元素按规则堆叠。**子元素的 z-index 再大也无法超出父层叠上下文的层级**。

**形成层叠上下文的常见条件**:
- 根元素 `<html>`;
- `position` 为 relative/absolute 且 `z-index` 不为 auto;
- `position: fixed / sticky`;
- `opacity` < 1;
- `transform`、`filter`、`will-change` 不为 none;
- flex/grid 容器的子项且 z-index 不为 auto。

### 3. 层叠顺序(从下到上)

背景/边框 → 负 z-index → block 块 → float → inline → z-index:auto/0 → 正 z-index。

## 代码示例

```css
/* absolute 相对最近的定位祖先 */
.parent { position: relative; }   /* 建立定位上下文 */
.child {
  position: absolute;
  top: 0; right: 0;               /* 相对 .parent 右上角 */
}

/* sticky 吸顶 */
.header {
  position: sticky;
  top: 0;                          /* 滚动到顶部时固定 */
  z-index: 10;
}
```

### z-index 失效经典案例

```css
/* 尽管 .a-child 的 z-index 是 9999,仍在 .b 下面 */
.a { position: relative; z-index: 1; }   /* 建立层叠上下文,层级 1 */
.a-child { position: relative; z-index: 9999; }
.b { position: relative; z-index: 2; }   /* 层级 2 */
/* 因为 a-child 的 9999 只在 .a 内部有效,.a 整体(层级1)< .b(层级2) */
```

## 高频追问

**Q:absolute 相对谁定位?**
A:相对最近的 `position` 非 static 的祖先元素;若没有,则相对初始包含块(视口)。

**Q:z-index 为什么有时不生效?**
A:① 元素未设 position(static 下 z-index 无效);② 比较的是不同层叠上下文,子元素 z-index 再大也受限于父层叠上下文的层级。

**Q:什么会创建层叠上下文?**
A:根元素、定位+z-index、fixed/sticky、opacity<1、transform/filter/will-change 不为 none、flex/grid 子项+z-index 等。

**Q:sticky 定位的原理和注意点?**
A:在跨越设定阈值(如 top:0)前表现为 relative,之后变 fixed。注意:父容器不能有 `overflow: hidden/auto`,否则失效;必须指定 top/bottom 等阈值。

**Q:fixed 相对谁?有什么例外?**
A:通常相对视口。例外:祖先若有 transform/filter/perspective 不为 none,fixed 会相对该祖先定位。

## 一句话背诵版

**position:relative 相对自身占位、absolute 相对最近定位祖先脱流、fixed 相对视口、sticky 滚动吸附;z-index 只在同一层叠上下文内比较,transform/opacity 等会创建新层叠上下文,导致子元素 z-index 再大也被父层级限制。**
