# C6 · 选择器与优先级

> 热度:🔥🔥

## 一句话考点

CSS 优先级按 **(行内, ID, 类/属性/伪类, 元素/伪元素)** 四元组从左到右比较,`!important` 最高;同级别按**后定义覆盖先定义**。

## 原理精讲

### 1. 常用选择器

- 基础:元素 `div`、类 `.cls`、ID `#id`、通配 `*`;
- 关系:后代 `A B`、子代 `A > B`、相邻兄弟 `A + B`、通用兄弟 `A ~ B`;
- 属性:`[type="text"]`、`[href^="https"]`(开头)、`[class$="btn"]`(结尾)、`[class*="col"]`(包含);
- 伪类:`:hover`、`:nth-child(n)`、`:first-child`、`:not()`、`:focus`;
- 伪元素:`::before`、`::after`、`::placeholder`。

### 2. 优先级(权重)计算

用四元组 **(a, b, c, d)** 表示:

| 位 | 含义 | 权重 |
|----|------|------|
| a | 行内样式 `style=""` | 1000 |
| b | ID 选择器 | 100 |
| c | 类、属性、伪类 | 10 |
| d | 元素、伪元素 | 1 |

比较时从高位到低位逐位比较,高位大者胜(**不进位**,11 个类也比不过 1 个 ID)。

### 3. 特殊规则

- `!important` 优先级最高(但应尽量避免);
- 通配符 `*`、`+`、`>` 等关系符**不增加**权重;
- `:not()` 本身不计权重,但其**参数**计;
- 权重相同时,后写的覆盖先写的(就近原则)。

## 代码示例

```css
/* 计算各自权重 */
#nav .list li          /* (0,1,1,1) = 111 */
.list li.active        /* (0,0,2,1) = 21  */
div#nav ul li a        /* (0,1,0,4) = 104 */
*                      /* (0,0,0,0) = 0   */
li:hover               /* (0,0,1,1) = 11  */
input[type="text"]     /* (0,0,1,1) = 11  */

/* 结论:#nav .list li (111) > div#nav ul li a (104) > .list li.active (21) */
```

```css
/* 就近覆盖:两条权重相同,后者生效 */
.btn { color: red; }
.btn { color: blue; }   /* 最终 blue */
```

## 高频追问

**Q:CSS 优先级如何计算?**
A:按 (行内 1000, ID 100, 类/属性/伪类 10, 元素/伪元素 1) 累加,从高位到低位比较;`!important` > 行内 > ID > 类 > 元素 > 继承 > 通配。

**Q:`!important` 应该用吗?**
A:尽量避免。它破坏级联规则、难以覆盖、增加维护成本。只在覆盖第三方库或工具类等极少数场景使用。

**Q:11 个 class 能覆盖 1 个 id 吗?**
A:不能。优先级不进位,ID 位(百位)只要有值就压过任意多个 class(十位)。

**Q:`:nth-child(2)` 和 `:nth-of-type(2)` 区别?**
A:nth-child 在所有兄弟中数第 2 个,若第 2 个不是目标类型则不匹配;nth-of-type 只在同类型元素中数第 2 个。

**Q:伪类和伪元素区别?**
A:伪类(单冒号 `:hover`)表示元素的特定状态;伪元素(双冒号 `::before`)创建文档中不存在的虚拟元素。伪元素一个选择器只能用一个。

## 一句话背诵版

**优先级 = (行内, ID, 类/属性/伪类, 元素) 从高位比、不进位,`!important` 最高;相同权重后者覆盖前者;通配符和关系符不加权;伪类是状态、伪元素是虚拟节点。**
