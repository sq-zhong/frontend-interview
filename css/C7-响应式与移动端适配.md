# C7 · 响应式与移动端适配

> 热度:🔥

## 一句话考点

响应式让页面适配不同屏幕,核心工具:**媒体查询 + 相对单位(rem/vw/%)+ viewport 元标签**;移动端还要处理 **1px 边框**和高清屏(DPR)问题。

## 原理精讲

### 1. viewport 设置(移动端必备)

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```
- `width=device-width`:布局视口等于设备宽度;
- `initial-scale=1`:初始缩放 1:1。

### 2. 常用单位

| 单位 | 相对于 | 场景 |
|------|--------|------|
| `px` | 绝对像素 | 固定尺寸 |
| `%` | 父元素 | 流式布局 |
| `em` | 当前元素字体大小 | 局部缩放 |
| `rem` | 根元素(html)字体大小 | 整体等比缩放 |
| `vw/vh` | 视口宽/高的 1% | 全屏、自适应 |
| `vmin/vmax` | 视口宽高的较小/大值 | 方向自适应 |

### 3. 三种主流适配方案

- **媒体查询**:按断点切换样式,适合 PC + 移动响应式站。
- **rem 方案**:JS 动态设置 html 的 font-size(如 `fontSize = clientWidth / 10`),元素用 rem,整体等比缩放。配合 flexible.js / postcss-pxtorem。
- **vw 方案**:直接用 vw 表示尺寸,`1vw = 视口宽 1%`,无需 JS,现代主流。

### 4. 1px 问题

高清屏(DPR=2/3)下,CSS 1px 会渲染成 2/3 物理像素,显得粗。解决:`transform: scaleY(0.5)` 伪元素、或 `border-image`、或 0.5px(部分设备支持)。

## 代码示例

### 媒体查询

```css
/* 移动优先:默认写移动端,再向上覆盖 */
.container { width: 100%; }

@media (min-width: 768px) {      /* 平板 */
  .container { width: 750px; }
}
@media (min-width: 1200px) {     /* 桌面 */
  .container { width: 1140px; }
}
```

### 1px 边框(伪元素 + scale)

```css
.border-1px {
  position: relative;
}
.border-1px::after {
  content: '';
  position: absolute;
  left: 0; bottom: 0;
  width: 100%;
  height: 1px;
  background: #ddd;
  transform: scaleY(0.5);        /* DPR=2 时压成半像素 */
  transform-origin: 0 0;
}
```

### clamp 实现流式字号(现代)

```css
/* 字号在 16px~24px 间随视口线性变化,无需媒体查询 */
h1 { font-size: clamp(16px, 4vw, 24px); }
```

## 高频追问

**Q:rem 和 em 区别?**
A:em 相对**当前元素**字体大小(会层层叠加,易失控);rem 相对**根元素 html** 字体大小(统一基准,更可控)。

**Q:rem 方案和 vw 方案怎么选?**
A:vw 无需 JS、更简洁,是现代首选;rem 兼容性更老、需 JS 或 postcss 配合动态设根字号。两者常结合 postcss 自动转换。

**Q:1px 边框问题怎么解决?**
A:高清屏物理像素多导致 1px 变粗。常用伪元素 + `transform: scaleY(0.5)`,或按 DPR 用媒体查询区分,或直接写 0.5px(新设备支持)。

**Q:移动优先和 PC 优先区别?**
A:移动优先用 `min-width` 从小屏往大屏加样式;PC 优先用 `max-width` 从大屏往小屏改。移动优先更推荐(渐进增强)。

**Q:vw/vh 有什么坑?**
A:移动端浏览器地址栏伸缩会导致 100vh 高度跳动;可用新单位 `dvh`(动态视口高度)解决。

## 一句话背诵版

**响应式 = viewport 元标签 + 媒体查询 + 相对单位;rem 相对根字号(需动态设置)、vw 相对视口(现代首选);1px 边框用伪元素 + scaleY(0.5);移动优先用 min-width 渐进增强。**
