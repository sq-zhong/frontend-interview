# D6 · 设计低代码 / 可视化搭建平台

> 系统设计专项 · 题目 → 需求澄清 → 架构 → 难点与取舍 → 追问

## 题目

设计一个低代码/可视化搭建平台,让用户拖拽组件生成页面(类似问卷、活动页、后台表单搭建)。

## 需求澄清

- **场景**:营销活动页 / 表单 / 后台 CRUD / 大屏?不同场景侧重不同;
- **用户**:开发者(半低代码)还是运营(纯拖拽)?
- **核心诉求**:拖拽编排、属性配置、实时预览、生成可用页面、可扩展组件。

## 整体架构

```
物料区(组件库) → 画布(拖拽编排) → 属性配置面板
                        ↓
              页面 Schema(JSON 描述)
                        ↓
              渲染引擎(Schema → 真实页面)
                        ↓
              出码 / 直接运行
```

**核心思想:一切皆 Schema**——用一份 JSON 描述整个页面(有哪些组件、层级、属性、样式、事件、数据源),编辑器改 Schema,渲染引擎读 Schema 生成页面。

## 核心模块设计

### 1. Schema 设计(最关键)

```json
{
  "type": "Page",
  "children": [
    {
      "id": "btn_1",
      "type": "Button",
      "props": { "text": "提交", "type": "primary" },
      "style": { "width": "100px" },
      "events": { "onClick": { "action": "submitForm" } }
    }
  ]
}
```
Schema 是编辑器和渲染引擎之间的**契约**,决定平台的能力边界。

### 2. 物料(组件)体系

- 每个可拖拽组件 = 渲染组件 + 配置描述(有哪些可配 props、类型、默认值);
- 组件需**可扩展**:第三方按规范注册新组件(物料协议)。

### 3. 拖拽编排(画布)

- HTML5 Drag API 或库(react-dnd);
- 拖入时往 Schema 插入节点,支持嵌套容器、排序、选中、删除。

### 4. 属性配置面板

- 根据选中组件的"配置描述"动态生成表单;
- 改表单 → 更新 Schema 对应节点 props → 画布实时重渲染(数据驱动)。

### 5. 渲染引擎

- 递归遍历 Schema,把每个 type 映射到真实组件并传入 props/style;
- 编辑态(可选中拖拽)和运行态(纯展示)共用一套渲染,靠开关区分。

### 6. 出码 vs 运行时渲染

- **运行时渲染**:直接把 Schema + 渲染引擎发布,线上读 Schema 渲染(灵活、可动态改);
- **出码**:把 Schema 编译成 React/Vue 源码(可二次开发、脱离平台)。

## 难点与取舍

**Q:核心难点是什么?**
A:① **Schema 设计**——要足够灵活描述各种页面,又不能太复杂;② 组件的**可扩展性**(物料协议);③ 事件和**数据源**编排(组件如何绑数据、联动);④ 编辑器体验(撤销重做、拖拽精度)。

**Q:出码和运行时渲染怎么选?**
A:运行时渲染灵活、改了立即生效、适合运营高频改的活动页;出码可脱离平台二次开发、性能可控、适合需要交付源码的场景。很多平台两者都支持。

**Q:怎么支持复杂逻辑(联动、校验)?**
A:纯拖拽难覆盖所有逻辑。方案:① 预设常见动作(提交、跳转、显隐联动)配置化;② 开"代码块"口子让开发者写少量 JS(低代码而非零代码);③ 表达式引擎处理简单联动。

## 追问

**Q:为什么"一切皆 Schema"?**
A:用 JSON 描述页面,让编辑(改 Schema)和渲染(读 Schema)解耦。Schema 可存储、传输、版本管理、跨端渲染,是低代码平台的核心抽象。

**Q:撤销/重做怎么实现?**
A:Schema 是不可变数据,每次操作产生新 Schema 快照,用栈保存历史,撤销/重做即在历史栈中前后移动(命令模式)。

**Q:如何保证拖拽性能?**
A:大页面 Schema 变化时避免全量重渲染,用 key + memo 只更新变化节点;拖拽用节流;复杂画布可虚拟化。

## 一句话总结

**低代码平台核心是"一切皆 Schema"(JSON 描述页面):物料(组件+配置描述)→ 拖拽编排改 Schema → 属性面板按配置描述动态生成、改 props → 渲染引擎递归 Schema 映射真实组件(编辑态/运行态共用);出码(编译成源码)vs 运行时渲染(读 Schema)按场景选;难点是 Schema 设计、物料可扩展、事件数据源编排、撤销重做(快照栈)。**
