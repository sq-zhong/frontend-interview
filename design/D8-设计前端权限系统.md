# D8 · 设计前端权限系统

> 系统设计专项 · 题目 → 需求澄清 → 架构 → 难点与取舍 → 追问

## 题目

设计一个前端权限系统,控制不同角色能访问哪些页面、看到哪些按钮、调哪些接口。

## 需求澄清

- **粒度**:页面级(路由)?菜单级?按钮级(操作)?数据级(只看自己的)?
- **模型**:RBAC(基于角色)还是更细的 ABAC(基于属性)?
- **核心原则**:**前端权限是体验优化,不是安全边界——真正的安全必须后端校验**。

## 权限三个层级

| 层级 | 控制 | 实现 |
|------|------|------|
| **路由级** | 能否进入某页面 | 动态路由 / 路由守卫 |
| **菜单级** | 侧边栏显示哪些菜单 | 根据权限过滤菜单数据 |
| **元素级** | 按钮/操作是否可见可用 | 权限指令 / 权限组件 |

## 核心设计

### 1. 权限数据来源

登录后从后端拿到当前用户的**角色 + 权限点列表**(如 `['user:add', 'user:delete']`),存到全局状态。

### 2. 路由权限(两种方案)

**方案 A:前端配置全量路由 + 守卫过滤**
```js
// 路由守卫:进入前检查权限
router.beforeEach((to, from, next) => {
  if (to.meta.permission && !hasPermission(to.meta.permission)) {
    next('/403');        // 无权限跳 403
  } else {
    next();
  }
});
```

**方案 B:后端返回路由表,前端动态生成(更安全)**
```js
// 登录后拉取该用户可访问的路由,动态 addRoute
const accessRoutes = await fetchUserRoutes();
accessRoutes.forEach(route => router.addRoute(route));
```

### 3. 菜单权限

根据权限过滤菜单配置,只渲染有权访问的菜单项(通常和路由表联动)。

### 4. 按钮/元素级权限

```js
// 权限指令(Vue)
app.directive('permission', {
  mounted(el, binding) {
    if (!hasPermission(binding.value)) {
      el.parentNode?.removeChild(el);   // 无权限则移除元素
    }
  },
});
// 用法:<button v-permission="'user:delete'">删除</button>
```

```jsx
// 权限组件(React)
function Authorized({ permission, children }) {
  return hasPermission(permission) ? children : null;
}
// <Authorized permission="user:delete"><DeleteBtn /></Authorized>
```

## 难点与取舍

**Q:前端做了权限控制,还需要后端校验吗?**
A:**必须**。前端权限只是**体验优化**(不显示无权的入口),但用户可以绕过前端(改代码、直接调接口)。**每个敏感操作和数据访问,后端都必须独立鉴权**。前端权限 ≠ 安全。

**Q:前端配置路由 vs 后端返回路由,怎么选?**
A:前端配置全量 + 守卫过滤:简单,但路由信息全暴露在前端代码里;后端返回动态路由:更安全(无权的路由前端根本没有)、权限变更无需发版,但实现复杂。中大型系统倾向后端动态路由。

**Q:按钮级权限如何优雅实现?**
A:封装权限指令(Vue)或权限组件/hook(React),传入权限点,内部判断是否有权决定渲染/移除/禁用。避免在业务代码里散落 `if (hasPermission)` 判断。

## 追问

**Q:RBAC 模型是什么?**
A:基于角色的访问控制。用户 → 角色 → 权限,用户通过角色获得权限。好处是解耦——改角色的权限,所有该角色用户自动生效,无需逐个配置用户。

**Q:权限变更后如何实时生效?**
A:① 重新登录/刷新拉取最新权限;② 用长连接/轮询推送权限变更通知,前端刷新权限数据和路由;③ 关键操作时后端实时校验兜底(即使前端缓存了旧权限,后端也会拦截)。

**Q:如何防止用户手动访问无权路由?**
A:路由守卫拦截跳 403 + 后端接口鉴权兜底。即使用户改前端绕过路由守卫,拿到的数据接口也会被后端拒绝,保证安全。

## 一句话总结

**前端权限三层:路由级(守卫过滤 or 后端返回动态路由)、菜单级(过滤菜单)、元素级(权限指令/组件按权限点渲染);数据来源是登录后拿角色+权限点(RBAC);核心铁律:前端权限只是体验优化,能被绕过,真正安全必须后端对每个操作独立鉴权。**
