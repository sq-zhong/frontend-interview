# D7 · 设计组件库 / 设计系统

> 系统设计专项 · 题目 → 需求澄清 → 架构 → 难点与取舍 → 追问

## 题目

设计一个企业级 UI 组件库(类似 Ant Design / Element Plus)。

## 需求澄清

- **范围**:基础组件(Button/Input)+ 业务组件?是否含设计规范(Design Tokens)?
- **诉求**:一致性、可复用、可定制(主题)、可访问性、好的文档、易维护。

## 整体架构

```
设计规范(Design Tokens:色板/间距/字号/圆角)
        ↓
基础组件(原子:Button/Input/Icon)
        ↓
复合组件(Form/Table/Modal)
        ↓
主题系统 + 文档站 + 按需引入 + 类型定义
```

## 核心设计点

### 1. 组件 API 设计(最能体现功力)

- **一致性**:同类 props 命名统一(size/type/disabled),降低学习成本;
- **受控 + 非受控**:如 Input 同时支持 `value`(受控)和 `defaultValue`(非受控);
- **组合优于配置**:复杂组件用子组件组合(`<Select><Option/></Select>`)而非一堆 props;
- **合理的默认值**:开箱即用;
- **灵活扩展**:透传 `className`/`style`/原生属性(`...rest`),暴露 ref。

### 2. 主题定制(Design Tokens)

- 用 **CSS 变量**定义 tokens(主色、圆角、间距),运行时可切换主题(亮/暗);
- 或 Sass 变量 + 编译期定制;
- 分层:全局 token → 组件级 token,便于精细定制。

### 3. 样式方案

- CSS-in-JS(动态主题好,有运行时开销)/ Sass + BEM(��典)/ CSS Modules / Tailwind;
- 需处理样式隔离和覆盖顺序。

### 4. 按需引入(Tree-shaking)

- ESM + `sideEffects` 配置,让未用组件被摇掉(见 E4);
- 每个组件独立入口,支持 `import { Button } from 'lib'` 只打包 Button。

### 5. 可访问性(a11y)

- 语义化标签、ARIA 属性、键盘导航(Tab/Enter/Esc)、焦点管理——企业级组件库的重要门槛。

### 6. 工程化配套

- 文档站(Storybook / dumi):组件示例 + API 表 + 可交互 demo;
- 完整 TS 类型定义;
- 单元测试 + 视觉回归测试;
- Monorepo 管理(见 E8)。

## 难点与取舍

**Q:组件 API 怎么设计得好用?**
A:一致性(统一命名)+ 受控/非受控都支持 + 组合优于堆 props + 合理默认值 + 可透传扩展。设计时从使用者视角出发,参考成熟库,避免过度设计。

**Q:如何支持换肤/主题?**
A:优先 CSS 变量定义 Design Tokens,运行时改变量即可切主题(支持动态、暗色模式),无需重新编译;配合分层 token(全局→组件级)支持精细定制。

**Q:如何做到按需引入?**
A:组件用 ESM 编写、每个独立入口、package.json 配 sideEffects,配合打包器 tree-shaking,让用户 `import { Button }` 时只打包用到的组件和样式。

## 追问

**Q:受控和非受控组件都支持,怎么实现?**
A:内部维护一个 state,若外部传了 value(受控)就用外部的、变化靠 onChange 通知;没传就用内部 state(非受控)+ defaultValue。用一个 `useControllableValue` hook 统一处理。

**Q:如何保证组件库质量和一致性?**
A:① 设计规范 + Design Tokens 统一视觉;② 统一的 API 约定;③ 单元测试 + 视觉回归;④ 文档站强制每个组件有示例和 API 说明;⑤ code review + 贡献规范。

**Q:CSS-in-JS 和 Sass 怎么选?**
A:CSS-in-JS 动态主题、作用域隔离好,但有运行时开销、SSR 复杂;Sass/CSS Modules 无运行时开销、成熟,但动态主题要靠 CSS 变量。看是否需要高度动态的主题能力。

## 一句话总结

**组件库设计:Design Tokens(CSS 变量)→ 原子/复合组件 → 主题+文档+按需引入;API 要一致命名、受控/非受控都支持、组合优于配置、可透传扩展;主题用 CSS 变量运行时切换;按需引入靠 ESM+sideEffects+tree-shaking;门槛是可访问性(a11y)和工程化(文档站/TS/测试/monorepo)。**
