# B2 · 事件机制(冒泡 / 捕获 / 委托)

> 热度:🔥🔥🔥

## 一句话考点

DOM 事件流分**捕获阶段(外→内)→ 目标阶段 → 冒泡阶段(内→外)**;利用冒泡把子元素事件统一交给父元素处理叫**事件委托**,能提升性能、支持动态元素。

## 原理精讲

### 1. 事件流三阶段

```
① 捕获阶段:window → document → ... → 目标父级(从外向内)
② 目标阶段:到达触发事件的元素
③ 冒泡阶段:目标 → ... → document → window(从内向外)
```

`addEventListener(type, fn, useCapture)`:
- 第三个参数 `false`(默认):**冒泡**阶段触发;
- `true`:**捕获**阶段触发。

### 2. 阻止与控制

| 方法 | 作用 |
|------|------|
| `e.stopPropagation()` | 阻止继续传播(冒泡/捕获) |
| `e.stopImmediatePropagation()` | 阻止传播 + 阻止同元素其他监听器 |
| `e.preventDefault()` | 阻止默认行为(如链接跳转、表单提交) |
| `return false`(仅内联/jQuery) | 同时阻止冒泡和默认(原生 addEventListener 中无效) |

### 3. 事件委托(Event Delegation)

把多个子元素的事件监听**绑到共同父元素**上,利用冒泡在父元素统一处理,通过 `e.target` 判断实际触发的子元素。

**优点**:
- **性能**:只绑一个监听器,而非给每个子元素绑(减少内存);
- **动态元素**:新增的子元素自动"拥有"事件,无需重新绑定;
- 代码更简洁。

### 4. target vs currentTarget

- `e.target`:实际触发事件的元素(冒泡的源头);
- `e.currentTarget`:当前绑定监听器的元素(= 事件委托里的父元素,等于 `this`)。

## 代码示例

### 事件委托

```js
// ❌ 传统:给每个 li 绑事件(1000 个 li = 1000 个监听器)
document.querySelectorAll('li').forEach(li => {
  li.addEventListener('click', handler);
});

// ✅ 事件委托:只在 ul 上绑一个
document.querySelector('ul').addEventListener('click', (e) => {
  // 判断是否点在 li 上(或用 closest 兼容内部元素)
  const li = e.target.closest('li');
  if (li) {
    console.log('点击了', li.textContent);
  }
});
// 之后动态 append 的新 li 也能响应,无需重新绑定
```

### 捕获 vs 冒泡顺序

```js
// 点击子元素,输出顺序:parent capture → child → parent bubble
parent.addEventListener('click', () => console.log('parent capture'), true);
child.addEventListener('click', () => console.log('child'));
parent.addEventListener('click', () => console.log('parent bubble'), false);
```

## 高频追问

**Q:事件流的三个阶段?**
A:捕获阶段(从 window 向下到目标的父级)、目标阶段(到达目标元素)、冒泡阶段(从目标向上回到 window)。默认监听在冒泡阶段,addEventListener 第三参数为 true 则在捕获阶段。

**Q:什么是事件委托?好处?**
A:利用冒泡,把子元素的事件统一绑定到父元素,用 e.target 判断实际触发者。好处:减少监听器数量节省内存、动态新增的子元素自动生效、代码更简洁。

**Q:target 和 currentTarget 区别?**
A:target 是实际触发事件的元素(冒泡源头);currentTarget 是当前处理事件、绑定监听器的元素(等于 this)。事件委托中 currentTarget 是父元素,target 是被点击的子元素。

**Q:stopPropagation 和 preventDefault 区别?**
A:stopPropagation 阻止事件继续在 DOM 树上传播(阻止冒泡/捕获);preventDefault 阻止元素的默认行为(链接跳转、表单提交、右键菜单),不影响传播。两者独立。

**Q:事件委托有什么局限?**
A:① 不冒泡的事件无法委托(如 focus、blur,需用 focusin/focusout 或捕获);② 需要精确判断 target(内部有嵌套元素时用 closest);③ 大量层级深的委托判断可能有额外开销。

## 一句话背诵版

**事件流:捕获(外→内)→目标→冒泡(内→外),addEventListener 第三参 true 为捕获;事件委托利用冒泡把子元素事件绑到父元素、用 e.target 判断,省内存且支持动态元素;target 是触发源、currentTarget 是绑定者;stopPropagation 阻传播、preventDefault 阻默认行为。**
