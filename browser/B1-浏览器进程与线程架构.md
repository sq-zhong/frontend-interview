# B1 · 浏览器进程与线程架构

> 热度:🔥🔥🔥

## 一句话考点

现代浏览器是**多进程**架构(浏览器主进程、渲染进程、GPU、网络等);每个标签页通常一个**渲染进程**,其中**渲染主线程**单线程执行 JS + 渲染,还有 GUI、事件、网络、定时器等多个线程配合。

## 原理精讲

### 1. 浏览器的多进程

| 进程 | 职责 |
|------|------|
| **浏览器主进程(Browser)** | 地址栏、书签、前进后退、管理其他进程 |
| **渲染进程(Renderer)** | 页面渲染、执行 JS、事件处理(每个 tab 一般独立,受沙箱隔离) |
| **网络进程(Network)** | 处理网络请求 |
| **GPU 进程** | 3D 绘制、合成加速 |
| **插件进程** | 运行插件 |

**多进程好处**:① 一个 tab 崩溃不影响其他;② 安全沙箱隔离;③ 充分利用多核。

### 2. 渲染进程中的主要线程

| 线程 | 职责 |
|------|------|
| **GUI 渲染线程** | 解析 HTML/CSS、构建 DOM/渲染树、布局、绘制 |
| **JS 引擎线程(主线程)** | 执行 JS |
| **事件触发线程** | 把满足条件的回调(事件、定时器到期)放入任务队列 |
| **定时器线程** | 计时(setTimeout/setInterval 的计时不占主线程) |
| **异步 HTTP 请求线程** | 处理 XHR/fetch,完成后把回调加入队列 |

### 3. 关键点:GUI 与 JS 互斥

**GUI 渲染线程与 JS 引擎线程互斥**。JS 执行时 GUI 线程被挂起,渲染暂停。所以:
- 长时间运行的 JS 会**阻塞页面渲染**(卡顿);
- 这也是耗时计算要放 Web Worker 的原因(见 B6)。

### 4. 为什么 JS 是单线程

JS 设计为单线程是为了避免多线程同时操作 DOM 的竞态问题(比如一个线程删节点、另一个改它)。用事件循环 + 异步实现非阻塞(见 J6)。

## 代码示例

```js
// 定时器计时由"定时器线程"处理,不占用主线程
// 但回调仍要等主线程空闲才执行
setTimeout(() => console.log('到期'), 1000);

// 长任务阻塞渲染:这段同步循环期间页面卡死、无法响应点击
const start = Date.now();
while (Date.now() - start < 3000) {}   // ❌ 阻塞主线程 3 秒

// 解决:切片(时间分片)或放 Web Worker
function chunkedWork(items, index = 0) {
  const end = Math.min(index + 100, items.length);
  for (let i = index; i < end; i++) process(items[i]);
  if (end < items.length) {
    // 让出主线程,给渲染/交互机会
    requestAnimationFrame(() => chunkedWork(items, end));
  }
}
```

## 高频追问

**Q:浏览器为什么用多进程?**
A:① 稳定性——单个 tab/插件崩溃不拖垮整个浏览器;② 安全——沙箱隔离,渲染进程无法直接访问系统资源;③ 性能——利用多核并行。代价是内存占用更高。

**Q:一个页面有哪些线程?**
A:渲染进程内主要有:GUI 渲染线程、JS 引擎线程(主线程)、事件触发线程、定时器线程、异步请求线程。JS 主线程单线程,其余线程配合把回调塞进任务队列。

**Q:为什么 JS 执行会阻塞页面渲染?**
A:因为 GUI 渲染线程和 JS 引擎线程互斥,不能同时运行。JS 长时间占用主线程时,渲染被挂起,页面无法更新和响应交互,表现为卡顿甚至假死。

**Q:setTimeout 的计时准确吗?**
A:计时由独立的定时器线程处理,但到期后的回调要等主线程空闲、且微任务清空后才执行。若主线程繁忙,实际执行会延迟,所以不精确。

**Q:如何避免长任务卡住页面?**
A:① 时间分片(把大任务拆成小块,用 requestIdleCallback/requestAnimationFrame/setTimeout 分批,让出主线程);② 放 Web Worker 在后台线程计算;③ 虚拟列表减少 DOM 量。

## 一句话背诵版

**浏览器多进程(主/渲染/网络/GPU),每 tab 一个渲染进程(崩溃隔离+沙箱+多核);渲染进程内:GUI 渲染线程、JS 主线程(单线程)、事件、定时器、异步请求线程;GUI 与 JS 互斥,所以长任务阻塞渲染,要用分片或 Web Worker。**
