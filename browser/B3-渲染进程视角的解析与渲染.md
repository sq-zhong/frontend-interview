# B3 · 从输入 URL 到渲染:渲染进程视角

> 热度:🔥🔥🔥

> 本篇聚焦**渲染进程内部**的解析与渲染细节,与 H2(全链路网络)、H3(重排重绘)互补。

## 一句话考点

渲染进程拿到 HTML 后:**解析 HTML 构建 DOM → 解析 CSS 构建 CSSOM → 合成渲染树 → 布局 → 分层 → 绘制 → 栅格化 → 合成显示**;JS 会阻塞解析、CSS 会阻塞渲染。

## 原理精讲

### 1. 完整渲染管线

```
HTML ──解析──> DOM 树
CSS  ──解析──> CSSOM 树
         ↓ 合并
     Render Tree(渲染树,只含可见节点)
         ↓
     Layout(布局/回流):计算每个节点几何位置
         ↓
     Layer(分层):生成图层树
         ↓
     Paint(绘制):生成每层的绘制指令列表
         ↓
     Raster(栅格化):把图层块转成位图(GPU)
         ↓
     Composite(合成):合成各图层,显示到屏幕
```

### 2. 解析细节

- **DOM 构建**:边下载边解析,遇到标签生成节点;
- **CSSOM 构建**:CSS 是**渲染阻塞资源**——CSSOM 没构建完,渲染树无法生成(白屏);
- **预解析线程**:主线程解析时,预解析线程提前扫描并下载 CSS/JS/图片等资源。

### 3. JS 对解析的影响(关键)

- `<script>` 会**阻塞 DOM 解析**:遇到脚本,HTML 解析暂停,先下载并执行脚本(因为 JS 可能操作 DOM);
- 且 JS 执行前要**等待前面的 CSS 下载完**(因为 JS 可能读取样式);
- 优化:`defer`(解析完按序执行)、`async`(下载完即执行),见 H4。

### 4. 分层与合成(性能关键)

- 某些属性(`transform`、`opacity`、`will-change`、`position:fixed`)会让元素**提升为独立合成层**;
- 合成层的变化只需在**合成阶段**用 GPU 处理,**跳过布局和绘制**,非常高效;
- 这就是 transform 动画比改 left/top 流畅的原因(见 H3)。

## 代码示例

```html
<!-- CSS 放头部:尽早开始构建 CSSOM,避免无样式闪烁 -->
<head>
  <link rel="stylesheet" href="style.css">   <!-- 渲染阻塞 -->
  <link rel="preload" href="hero.jpg" as="image">  <!-- 预加载关键资源 -->
</head>
<body>
  <div id="app"></div>
  <!-- JS 放底部或用 defer,避免阻塞 DOM 解析 -->
  <script defer src="app.js"></script>
</body>
```

```css
/* 提升为合成层,动画走 GPU 合成,不触发布局/绘制 */
.card {
  will-change: transform;
}
.card:hover {
  transform: scale(1.05);   /* 只在合成阶段处理 */
}
```

## 高频追问

**Q:CSS 会阻塞什么?**
A:CSS 阻塞渲染(CSSOM 未完成则渲染树无法生成,页面白屏),但不阻塞 DOM 解析。此外,CSS 会间接阻塞其后 JS 的执行(JS 可能读样式,需等 CSSOM)。

**Q:为什么 script 会阻塞 DOM 解析?**
A:因为脚本可能修改 DOM 结构(document.write、增删节点),浏览器无法预知,只能暂停解析、先执行脚本以保证正确性。用 defer/async 可避免阻塞。

**Q:什么是合成层?有什么好处?**
A:某些元素(用了 transform/opacity/will-change/fixed 等)会被提升为独立图层。这些层的变化只在合成阶段由 GPU 处理,跳过布局和绘制,性能高、动画流畅。

**Q:DOM 树和渲染树的区别?**
A:DOM 树包含所有节点(含 display:none 的);渲染树只包含**可见节点**(display:none 不在渲染树,visibility:hidden 在但不可见),用于布局和绘制。

**Q:如何优化关键渲染路径?**
A:① CSS 尽早加载、精简关键 CSS(内联首屏样式);② JS 用 defer/async 避免阻塞;③ 减少渲染阻塞资源;④ preload/prefetch 关键资源;⑤ 减少 DOM 层级和重排。

## 一句话背诵版

**渲染进程:HTML→DOM、CSS→CSSOM,合成渲染树(仅可见节点)→布局→分层→绘制→栅格化→合成显示;CSS 阻塞渲染、script 阻塞 DOM 解析(可能操作 DOM)且等 CSSOM;transform/opacity/will-change 提升合成层,变化只走 GPU 合成、跳过布局绘制。**
