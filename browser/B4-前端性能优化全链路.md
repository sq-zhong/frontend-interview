# B4 · 前端性能优化全链路

> 热度:🔥🔥🔥

## 一句话考点

性能优化按"**加载 → 渲染 → 运行时**"三阶段展开:加载优化(减少体积、请求、用缓存 CDN)、渲染优化(关键路径、减少重排)、运行时优化(防抖节流、虚拟列表、Web Worker)。

## 原理精讲

### 1. 加载阶段

| 手段 | 说明 |
|------|------|
| **减小体积** | 代码压缩(Terser)、Gzip/Brotli、Tree-shaking、按需引入 |
| **减少请求** | 雪碧图、资源合并、HTTP/2 多路复用 |
| **代码分割** | 路由/组件懒加载(动态 import),首屏只加载必要代码 |
| **缓存** | 强缓存 + 协商缓存(见 N4),静态资源 hash + CDN |
| **图片优化** | WebP/AVIF、响应式图片、懒加载、雪碧图 |
| **预加载** | `preload`(当前页关键资源)、`prefetch`(未来页资源)、`dns-prefetch`/`preconnect` |

### 2. 渲染阶段

- 优化关键渲染路径:CSS 提前、JS defer/async(见 B3);
- 减少重排重绘:批量 DOM、读写分离、transform 动画(见 H3);
- 首屏:SSR/SSG、骨架屏、内联首屏 CSS;
- 避免无样式闪烁(FOUC)。

### 3. 运行时阶段

- **防抖节流**:高频事件(见 J10);
- **虚拟列表**:长列表只渲染可视区(react-window / vue-virtual-scroller);
- **Web Worker**:耗时计算放后台线程(见 B6);
- **时间分片**:大任务拆分让出主线程(requestIdleCallback);
- **事件委托**:减少监听器(见 B2)。

### 4. 核心指标(Web Vitals)

| 指标 | 含义 | 目标 |
|------|------|------|
| **LCP** | 最大内容绘制(加载) | < 2.5s |
| **INP** | 交互到下次绘制(响应,取代 FID) | < 200ms |
| **CLS** | 累积布局偏移(视觉稳定) | < 0.1 |
| FCP | 首次内容绘制 | < 1.8s |
| TTFB | 首字节时间 | 越小越好 |

## 代码示例

### 路由懒加载(代码分割)

```js
// React
const Home = lazy(() => import('./Home'));
<Suspense fallback={<Skeleton />}>
  <Home />
</Suspense>

// Vue Router
const routes = [
  { path: '/home', component: () => import('./Home.vue') },  // 按需加载
];
```

### 图片懒加载

```html
<!-- 原生懒加载 -->
<img src="thumb.jpg" loading="lazy" alt="">

<!-- IntersectionObserver 手动实现 -->
<script>
const io = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      const img = entry.target;
      img.src = img.dataset.src;   // 进入视口才加载
      io.unobserve(img);
    }
  });
});
document.querySelectorAll('img[data-src]').forEach(img => io.observe(img));
</script>
```

## 高频追问

**Q:前端性能优化从哪些方面入手?**
A:分三阶段——加载(压缩、Tree-shaking、代码分割、缓存、CDN、图片优化、预加载)、渲染(关键渲染路径、减少重排重绘、SSR/骨架屏)、运行时(防抖节流、虚拟列表、Web Worker、时间分片)。

**Q:如何优化首屏加载?**
A:代码分割只加载首屏 JS、SSR/SSG 直出、内联关键 CSS、路由/图片懒加载、preload 关键资源、CDN 加速、开启 Gzip/Brotli、骨架屏改善感知。

**Q:核心 Web Vitals 有哪些?**
A:LCP(最大内容绘制,加载性能,<2.5s)、INP(交互响应,<200ms,已取代 FID)、CLS(累积布局偏移,视觉稳定,<0.1)。是 Google 衡量用户体验的核心指标。

**Q:长列表怎么优化?**
A:虚拟列表——只渲染可视区域内的 DOM,滚动时动态替换内容,把上千条数据的 DOM 量降到几十个,大幅减少渲染和内存开销。

**Q:如何减少 HTTP 请求?**
A:资源合并/雪碧图、内联小资源(base64)、HTTP/2 多路复用、按需加载合并小接口、利用缓存避免重复请求。

## 一句话背诵版

**性能优化三阶段:加载(压缩/Tree-shaking/代码分割/缓存 CDN/图片懒加载/preload)、渲染(关键渲染路径/减少重排/SSR 骨架屏)、运行时(防抖节流/虚拟列表/Web Worker/时间分片);核心指标 LCP<2.5s、INP<200ms、CLS<0.1。**
