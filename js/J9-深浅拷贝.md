# J9 · 深浅拷贝

> 热度:🔥🔥

## 一句话考点

浅拷贝只复制第一层,嵌套对象仍共享引用;深拷贝递归复制所有层级,得到完全独立的副本;手写深拷贝要处理**循环引用**和特殊类型。

## 原理精讲

### 1. 赋值 vs 浅拷贝 vs 深拷贝

- **赋值**:只拷贝引用地址,改一个另一个跟着变。
- **浅拷贝**:新对象第一层是独立的,但嵌套的引用类型仍指向同一地址。
- **深拷贝**:递归复制,新旧对象完全独立,互不影响。

### 2. 浅拷贝方式

- `Object.assign({}, obj)`
- 扩展运算符 `{...obj}`、`[...arr]`
- 数组 `slice()`、`concat()`

### 3. 深拷贝方式

| 方式 | 优点 | 缺点 |
|------|------|------|
| `JSON.parse(JSON.stringify())` | 简单 | 丢失 undefined/函数/Symbol;Date 变字符串;不能处理循环引用;正则变空对象 |
| `structuredClone()`(现代浏览器) | 原生、支持循环引用/Date/Map/Set | 不支持函数;兼容性需注意 |
| 手写递归 | 可控 | 需自己处理边界 |

## 代码示例

### JSON 方法的坑

```js
const obj = {
  a: undefined,
  b: () => {},
  c: new Date(),
  d: NaN,
};
JSON.parse(JSON.stringify(obj));
// { c: '2026-...(字符串)', d: null }   ← a、b 丢失,c 变字符串,NaN 变 null
```

### 手写深拷贝(处理循环引用 + 常见类型)

```js
function deepClone(target, map = new WeakMap()) {
  // 原始类型和函数直接返回
  if (target === null || typeof target !== 'object') return target;

  // 处理特殊对象
  if (target instanceof Date) return new Date(target);
  if (target instanceof RegExp) return new RegExp(target);

  // 循环引用:已拷贝过就直接返回
  if (map.has(target)) return map.get(target);

  // 保持原型
  const clone = Array.isArray(target)
    ? []
    : Object.create(Object.getPrototypeOf(target));
  map.set(target, clone);

  // Map / Set
  if (target instanceof Map) {
    target.forEach((v, k) => clone.set(deepClone(k, map), deepClone(v, map)));
    return clone;
  }
  if (target instanceof Set) {
    target.forEach(v => clone.add(deepClone(v, map)));
    return clone;
  }

  // 普通对象/数组(含 Symbol 键)
  Reflect.ownKeys(target).forEach(key => {
    clone[key] = deepClone(target[key], map);
  });
  return clone;
}
```

### 验证循环引用

```js
const a = { name: 'a' };
a.self = a;                     // 循环引用
const b = deepClone(a);
b.self === b;                   // true(正确处理,不会栈溢出)
b !== a;                        // true
```

## 高频追问

**Q:为什么用 WeakMap 而不是 Map?**
A:WeakMap 的键是弱引用,拷贝结束后自动被垃圾回收,避免内存泄漏;而且用来记录"原对象→拷贝对象"的映射,天然适合对象键。

**Q:`JSON.parse(JSON.stringify())` 有哪些问题?**
A:① undefined、函数、Symbol 会丢失;② Date 变成字符串;③ 循环引用直接报错;④ 正则、Map、Set 变空对象/丢失;⑤ NaN、Infinity 变 null。

**Q:如何深拷贝一个含函数的对象?**
A:JSON 方法不行。用手写递归(函数直接返回引用,通常无需拷贝函数)或第三方库 lodash 的 `cloneDeep`。

**Q:扩展运算符是深拷贝吗?**
A:不是,只是浅拷贝第一层。`{...obj}` 的嵌套对象仍共享引用。

## 一句话背诵版

**浅拷贝复制第一层(`Object.assign`/扩展运算符),嵌套仍共享;深拷贝递归独立;JSON 法简单但丢函数/undefined、不处理循环引用;手写深拷贝用 WeakMap 解决循环引用、递归处理 Date/RegExp/Map/Set。**
