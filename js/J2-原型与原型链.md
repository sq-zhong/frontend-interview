# J2 · 原型与原型链

> 热度:🔥🔥🔥

## 一句话考点

每个对象都有 `__proto__` 指向其构造函数的 `prototype`,层层向上串成**原型链**,查找属性时沿链向上找,是 JS 继承的底层机制。

## 原理精讲

### 1. 三个核心概念

- **`prototype`**:函数才有的属性,指向原型对象。
- **`__proto__`**(即 `[[Prototype]]`):每个对象都有,指向其构造函数的 `prototype`。
- **`constructor`**:原型对象上的属性,指回构造函数。

三者关系:
```
实例.__proto__ === 构造函数.prototype
构造函数.prototype.constructor === 构造函数
```

### 2. 原型链

访问对象属性时:
1. 先在对象自身找;
2. 找不到就沿 `__proto__` 到原型对象找;
3. 再找不到继续沿原型的 `__proto__` 向上;
4. 直到 `Object.prototype`,它的 `__proto__` 是 `null`,查找结束返回 `undefined`。

```
实例 → 构造函数.prototype → Object.prototype → null
```

### 3. 关键结论(必背)

```js
Object.prototype.__proto__ === null          // 原型链顶端
Function.prototype.__proto__ === Object.prototype
Object.__proto__ === Function.prototype       // Object 本身是函数
Function.__proto__ === Function.prototype     // 经典:函数由 Function 创建
```

## 代码示例

```js
function Person(name) {
  this.name = name;
}
Person.prototype.sayName = function () {
  console.log(this.name);
};

const p = new Person('Tom');
p.sayName();                                  // 'Tom'(在原型上找到)

p.__proto__ === Person.prototype;             // true
Person.prototype.constructor === Person;      // true
p.__proto__.__proto__ === Object.prototype;   // true
p.hasOwnProperty('name');                     // true(自身)
p.hasOwnProperty('sayName');                  // false(原型上)
```

### 六种继承方式演进(重点)

```js
// 1. 原型链继承 —— 缺点:引用属性被所有实例共享
Child.prototype = new Parent();

// 2. 构造函数继承 —— 缺点:无法继承原型方法
function Child() { Parent.call(this); }

// 3. 组合继承(最常用) —— 缺点:调用两次 Parent
function Child(name) {
  Parent.call(this, name);            // 第二次
}
Child.prototype = new Parent();        // 第一次
Child.prototype.constructor = Child;

// 4. 寄生组合继承(最优,ES5 最佳实践)
function Child(name) {
  Parent.call(this, name);
}
Child.prototype = Object.create(Parent.prototype); // 不调用构造函数
Child.prototype.constructor = Child;

// 5. class extends(ES6,底层就是寄生组合)
class Child extends Parent {
  constructor(name) {
    super(name);   // 必须先调 super
  }
}
```

## 高频追问

**Q:`new` 做了什么?**
A:① 创建空对象;② 空对象的 `__proto__` 指向构造函数 `prototype`;③ 执行构造函数,`this` 指向新对象;④ 若构造函数返回对象则返回它,否则返回新对象。

**Q:`Object.create(null)` 有什么特点?**
A:创建一个没有原型的纯净对象,没有 `toString`、`hasOwnProperty` 等继承方法,常用作字典避免原型污染。

**Q:`class` 和构造函数的区别?**
A:class 本质是语法糖,但:① 不会变量提升;② 内部默认严格模式;③ 方法不可枚举;④ 必须 `new` 调用;⑤ 子类必须调 `super`。

**Q:如何判断属性是自身的还是原型的?**
A:`obj.hasOwnProperty(key)` 只判自身;`key in obj` 会顺原型链查找。

## 一句话背诵版

**实例的 `__proto__` 指向构造函数的 `prototype`,层层向上到 `Object.prototype` 再到 `null`;查属性沿链向上找;继承首选寄生组合(`Object.create`)或 ES6 `class extends`。**
