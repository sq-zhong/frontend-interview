# J10 · 防抖与节流

> 热度:🔥🔥

## 一句话考点

都是**限制高频函数执行**的优化手段:**防抖(debounce)**——事件停止 N 毫秒后才执行(只认最后一次);**节流(throttle)**——每隔 N 毫秒最多执行一次(固定频率)。

## 原理精讲

### 防抖 debounce

在事件被触发 N 秒后再执行回调,如果 N 秒内又被触发,则**重新计时**。适合"只关心最终状态"的场景。

**场景**:搜索框输入联想、窗口 resize、表单校验、按钮防重复提交。

### 节流 throttle

规定在 N 秒内只执行一次,无论触发多少次。适合"匀速响应"的场景。

**场景**:滚动加载(scroll)、鼠标移动、拖拽、高频点击。

### 形象比喻

- 防抖:电梯门,不断有人进就一直等,没人进了才关门。
- 节流:水龙头,固定速度滴水,拧再大也这么快。

## 代码示例

### 防抖(支持立即执行)

```js
function debounce(fn, delay, immediate = false) {
  let timer = null;
  return function (...args) {
    if (timer) clearTimeout(timer);

    if (immediate && !timer) {
      fn.apply(this, args);           // 首次立即执行
    }
    timer = setTimeout(() => {
      timer = null;
      if (!immediate) fn.apply(this, args);
    }, delay);
  };
}

// 用法
const onSearch = debounce(query => fetchSuggest(query), 300);
input.addEventListener('input', e => onSearch(e.target.value));
```

### 节流(时间戳 + 定时器,保证首尾都执行)

```js
function throttle(fn, interval) {
  let last = 0;
  let timer = null;
  return function (...args) {
    const now = Date.now();
    const remaining = interval - (now - last);
    if (remaining <= 0) {
      // 时间到,立即执行
      if (timer) { clearTimeout(timer); timer = null; }
      last = now;
      fn.apply(this, args);
    } else if (!timer) {
      // 保证最后一次也执行
      timer = setTimeout(() => {
        last = Date.now();
        timer = null;
        fn.apply(this, args);
      }, remaining);
    }
  };
}

// 用法
window.addEventListener('scroll', throttle(onScroll, 200));
```

## 高频追问

**Q:防抖和节流的核心区别?**
A:防抖是"延迟执行,重复触发就重新计时,最终只执行一次";节流是"固定间隔执行,不管触发多频繁,匀速执行"。

**Q:什么场景用防抖,什么用节流?**
A:防抖——输入联想、resize、防重复提交(只要最终结果);节流——滚动、拖拽、动画(要持续但匀速响应)。

**Q:防抖如何实现"首次立即执行"?**
A:加 immediate 参数,首次触发且无定时器时立即执行 fn,之后靠定时器控制。

**Q:节流有哪两种实现?**
A:① 时间戳版(首次立即执行,末次不执行);② 定时器版(首次延迟,末次执行);结合两者可保证首尾都执行。

**Q:lodash 的防抖节流有什么额外能力?**
A:支持 `leading`/`trailing` 配置首尾是否执行、`maxWait` 最大等待时间、`cancel`/`flush` 取消或立即触发。

## 一句话背诵版

**防抖:停止触发 N 毫秒后才执行,重复触发就重新计时,用于输入联想/防重复提交;节流:每 N 毫秒最多执行一次,匀速响应,用于滚动/拖拽;实现分别靠 setTimeout 重置 和 时间戳/定时器控制间隔。**
