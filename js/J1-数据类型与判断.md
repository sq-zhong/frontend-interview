# J1 · 数据类型与判断

> 热度:🔥🔥

## 一句话考点

JS 有 **8 种数据类型**(7 原始 + 1 引用),原始类型存栈、引用类型存堆;判断类型要根据场景选 `typeof` / `instanceof` / `Object.prototype.toString`。

## 原理精讲

### 1. 八种数据类型

**原始类型(Primitive)**:`undefined`、`null`、`boolean`、`number`、`string`、`symbol`(ES6)、`bigint`(ES2020)。

**引用类型(Reference)**:`object`(包括 `Array`、`Function`、`Date`、`RegExp` 等)。

区别:
- 原始类型**按值存储**在栈内存,复制时是值拷贝,互不影响。
- 引用类型**值存堆内存,引用地址存栈**,复制时拷贝的是地址,指向同一对象。

### 2. 三种判断方式对比

| 方式 | 能力 | 缺陷 |
|------|------|------|
| `typeof` | 快速区分原始类型 + function | `typeof null === 'object'`(历史 bug);数组/对象无法区分 |
| `instanceof` | 判断引用类型(基于原型链) | 无法判断原始类型;跨 iframe 失效 |
| `Object.prototype.toString.call()` | **最准确**,能区分所有类型 | 写法略繁琐 |

### 3. 经典陷阱

- `typeof null === 'object'`:因为 null 的机器码前三位为 0,被误判为对象。
- `NaN`:`typeof NaN === 'number'`,且 `NaN !== NaN`,判断要用 `Number.isNaN()`。
- `typeof function(){} === 'function'`:function 是唯一被 typeof 单独识别的引用类型。

## 代码示例

```js
// typeof
typeof 1;            // 'number'
typeof 'a';          // 'string'
typeof true;         // 'boolean'
typeof undefined;    // 'undefined'
typeof null;         // 'object'   ← 陷阱
typeof Symbol();     // 'symbol'
typeof 1n;           // 'bigint'
typeof function(){}; // 'function'
typeof [];           // 'object'   ← 无法区分数组

// instanceof(基于原型链)
[] instanceof Array;   // true
[] instanceof Object;  // true(Array.prototype 的原型是 Object.prototype)

// 最可靠:toString
const type = v => Object.prototype.toString.call(v).slice(8, -1).toLowerCase();
type([]);        // 'array'
type(null);      // 'null'
type(/a/);       // 'regexp'
type(new Date);  // 'date'
type(() => {});  // 'function'

// NaN 判断
Number.isNaN(NaN);   // true
Number.isNaN('a');   // false(不会隐式转换,优于全局 isNaN)
```

### 手写 instanceof

```js
function myInstanceof(left, right) {
  if (typeof left !== 'object' || left === null) return false;
  let proto = Object.getPrototypeOf(left);
  const prototype = right.prototype;
  while (true) {
    if (proto === null) return false;
    if (proto === prototype) return true;
    proto = Object.getPrototypeOf(proto);
  }
}
```

## 高频追问

**Q:`typeof` 和 `instanceof` 区别?**
A:typeof 返回字符串、擅长原始类型且性能好;instanceof 基于原型链查找、只能判断引用类型,跨执行环境(iframe)会失效因为原型不同。

**Q:如何准确判断数组?**
A:优先 `Array.isArray()`;兜底用 `Object.prototype.toString.call(x) === '[object Array]'`。

**Q:`0.1 + 0.2 !== 0.3` 为什么?**
A:IEEE 754 双精度浮点用二进制存储,0.1/0.2 无法精确表示,相加有精度误差。解决:`Math.abs(a+b-c) < Number.EPSILON`,或转整数运算。

**Q:`null` 和 `undefined` 区别?**
A:undefined 表示"未定义"(声明未赋值);null 表示"空对象指针"(主动赋空)。`null == undefined` 为 true,`null === undefined` 为 false。

## 一句话背诵版

**7 原始(undefined/null/boolean/number/string/symbol/bigint)+ 1 引用;typeof 判原始、Array.isArray 判数组、`toString.call` 判一切;typeof null 是 object、NaN 不等于自身。**
