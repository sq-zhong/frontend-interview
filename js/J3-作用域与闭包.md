# J3 · 作用域与闭包

> 热度:🔥🔥🔥

## 一句话考点

作用域决定变量的可访问范围,JS 用**词法(静态)作用域**;**闭包 = 函数 + 其定义时所处的词法环境**,能让函数记住并访问外层作用域的变量。

## 原理精讲

### 1. 作用域类型

- **全局作用域**:最外层。
- **函数作用域**:`var` 声明的变量以函数为边界。
- **块级作用域**(ES6):`let`/`const` 以 `{}` 为边界。

**词法作用域**:作用域在**代码书写时**就确定,取决于函数定义的位置,而非调用的位置。

### 2. 作用域链

内层函数可以访问外层变量,查找变量时从当前作用域逐层向外,直到全局,找不到则报 `ReferenceError`。这条链在函数**定义时**确定。

### 3. 闭包

**定义**:内部函数引用了外部函数的变量,使得外部函数执行完后,其变量仍不被回收——形成闭包。

**产生条件**:函数嵌套 + 内层函数引用外层变量 + 内层函数被外部持有。

**用途**:数据私有化、函数柯里化、防抖节流、模块化。

**代价**:被引用的变量常驻内存,用不好会内存泄漏。

## 代码示例

### 经典循环问题

```js
// 问题:输出 3 3 3(var 无块级作用域,共享同一个 i)
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}

// 解法1:let(每次迭代产生新的块级绑定)
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);  // 0 1 2
}

// 解法2:IIFE 闭包保存当前值
for (var i = 0; i < 3; i++) {
  (function (j) {
    setTimeout(() => console.log(j), 0); // 0 1 2
  })(i);
}
```

### 闭包做数据私有

```js
function createCounter() {
  let count = 0;                 // 外部无法直接访问
  return {
    inc: () => ++count,
    get: () => count,
  };
}
const c = createCounter();
c.inc();       // 1
c.get();       // 1
// count 被闭包保护,只能通过方法操作
```

## 高频追问

**Q:闭包一定会内存泄漏吗?**
A:不会。闭包只是让变量常驻内存,这是设计意图。泄漏是指"不再需要却无法释放",比如闭包引用了大对象且长期持有。用完置 `null` 可释放。

**Q:`let`/`const` 和 `var` 区别?**
A:① var 函数作用域、let/const 块级;② var 有变量提升(值为 undefined),let/const 有"暂时性死区"(TDZ),声明前访问报错;③ var 可重复声明,let/const 不可;④ const 声明后不可重新赋值(但对象内部可变)。

**Q:什么是暂时性死区(TDZ)?**
A:let/const 声明的变量在声明前不可访问,从块开始到声明处这段区域即 TDZ,访问会抛 `ReferenceError`。

**Q:说一个闭包的实际应用?**
A:防抖/节流(用闭包保存定时器)、柯里化、模块模式(暴露公有 API、隐藏私有变量)、React 的 useState(闭包保存 state)。

## 一句话背诵版

**作用域是词法的、定义时确定;闭包是函数携带其定义处的作用域,让外层变量在函数外仍可访问;循环用 `let` 或 IIFE 解决共享变量问题。**
