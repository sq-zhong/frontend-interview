# H1 · 语义化标签与文档结构

> 热度:🔥🔥

## 一句话考点

语义化就是**用正确的标签描述内容含义**(而非只为样式);好处是利于 SEO、可访问性(屏幕阅读器)、代码可维护性。

## 原理精讲

### 1. 什么是语义化

用带有明确含义的标签标记内容,让机器和人都能理解结构。比如用 `<nav>` 而非 `<div class="nav">`。

### 2. HTML5 语义化标签

| 标签 | 含义 |
|------|------|
| `<header>` | 页头/区块头部 |
| `<nav>` | 导航 |
| `<main>` | 主内容(一个页面一个) |
| `<article>` | 独立完整的内容(文章、评论) |
| `<section>` | 主题性内容分区 |
| `<aside>` | 侧边栏/附属内容 |
| `<footer>` | 页脚 |
| `<figure>`/`<figcaption>` | 图片及其说明 |
| `<time>` | 时间 |
| `<mark>` | 高亮标记 |

### 3. 语义化的好处

1. **SEO**:搜索引擎能更好理解页面结构和重点内容;
2. **可访问性**:屏幕阅读器可依据标签为视障用户导航;
3. **可维护性**:结构清晰,团队协作易读;
4. **无 CSS 时**:页面仍有合理的层次结构。

## 代码示例

```html
<body>
  <header>
    <h1>网站标题</h1>
    <nav>
      <ul>
        <li><a href="/">首页</a></li>
        <li><a href="/about">关于</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <header>
        <h2>文章标题</h2>
        <time datetime="2026-07-16">2026年7月16日</time>
      </header>
      <section>
        <h3>第一节</h3>
        <p>正文内容……</p>
      </section>
    </article>

    <aside>
      <h3>相关推荐</h3>
    </aside>
  </main>

  <footer>
    <p>&copy; 2026 版权所有</p>
  </footer>
</body>
```

## 高频追问

**Q:为什么要语义化?**
A:利于 SEO(搜索引擎理解内容)、提升可访问性(屏幕阅读器)、代码可读易维护、无样式时结构依然清晰。

**Q:`<section>` 和 `<article>` 区别?**
A:article 是**独立可复用**的完整内容(如一篇文章、一条评论,拿出去也成立);section 是**主题性分区**,通常是文档的一部分,需要标题。

**Q:`<b>` 和 `<strong>`、`<i>` 和 `<em>` 区别?**
A:b/i 只是视觉加粗/倾斜,无语义;strong 表示"重要",em 表示"强调",有语义,屏幕阅读器会体现。优先用后者。

**Q:一个页面能有多个 `<h1>` 吗?**
A:HTML5 允许每个 sectioning 区块有自己的 h1,但从 SEO 和实践角度,通常建议一个页面一个主 h1。

**Q:`<div>` 和 `<span>` 是语义化标签吗?**
A:不是,它们是无语义的通用容器(div 块级、span 行内),仅在没有合适语义标签时作为样式/布局钩子使用。

## 一句话背诵版

**语义化 = 用正确标签表达内容含义(header/nav/main/article/section/aside/footer);好处是 SEO、可访问性、可维护性;strong/em 有语义优于 b/i;div/span 无语义只作容器。**
