# H5 · 浏览器存储方案对比

> 热度:🔥🔥

## 一句话考点

前端存储主要有 **cookie、localStorage、sessionStorage、IndexedDB**;区别在容量、生命周期、是否随请求发送、作用域和 API 类型。

## 原理精讲

### 四种存储对比

| 特性 | cookie | localStorage | sessionStorage | IndexedDB |
|------|--------|-------------|----------------|-----------|
| 容量 | ~4KB | ~5-10MB | ~5-10MB | 大(几百 MB+) |
| 生命周期 | 可设过期时间 | 永久(手动清除) | 会话级(关闭标签页清除) | 永久 |
| 随请求发送 | **是**(自动带在 header) | 否 | 否 | 否 |
| 作用域 | 域名+路径 | 同源 | 同源 + 同标签页 | 同源 |
| API | 字符串,操作繁琐 | 同步 kv | 同步 kv | 异步、事务型 |
| 存储类型 | 字符串 | 字符串 | 字符串 | 结构化(对象/二进制) |

### 各自适用场景

- **cookie**:身份认证(session id、token)、需随请求发给服务器的少量数据;
- **localStorage**:长期保存的客户端数据(主题、用户设置、非敏感缓存);
- **sessionStorage**:单次会话临时数据(表单草稿、单页流程中间态);
- **IndexedDB**:大量结构化数据、离线应用、缓存接口数据。

### cookie 关键属性

- `Expires`/`Max-Age`:过期时间(不设为会话 cookie);
- `Domain`/`Path`:作用范围;
- `HttpOnly`:禁止 JS 读取,防 XSS 窃取;
- `Secure`:仅 HTTPS 传输;
- `SameSite`:防 CSRF(Strict/Lax/None)。

## 代码示例

```js
// localStorage / sessionStorage(API 相同)
localStorage.setItem('theme', 'dark');
localStorage.getItem('theme');        // 'dark'
localStorage.removeItem('theme');
localStorage.clear();

// 存对象需序列化
localStorage.setItem('user', JSON.stringify({ name: 'Tom' }));
const user = JSON.parse(localStorage.getItem('user'));

// cookie(原生 API 繁琐)
document.cookie = 'token=abc; max-age=3600; path=/; Secure; SameSite=Lax';

// IndexedDB(异步)
const req = indexedDB.open('myDB', 1);
req.onupgradeneeded = e => {
  const db = e.target.result;
  db.createObjectStore('users', { keyPath: 'id' });
};
req.onsuccess = e => {
  const db = e.target.result;
  const tx = db.transaction('users', 'readwrite');
  tx.objectStore('users').add({ id: 1, name: 'Tom' });
};
```

## 高频追问

**Q:cookie、localStorage、sessionStorage 区别?**
A:核心三点——① 容量:cookie 4KB,其余 5-10MB;② 生命周期:cookie 可设过期、localStorage 永久、sessionStorage 关标签页即失效;③ cookie 每次请求自动携带(增加流量),其余仅本地。

**Q:token 存哪里更安全?**
A:各有取舍。存 cookie 配 HttpOnly + Secure + SameSite 可防 XSS 读取和一定 CSRF;存 localStorage 易被 XSS 读取但不受 CSRF 影响。实践中常用 HttpOnly cookie 存 refresh token。

**Q:localStorage 能跨标签页通信吗?**
A:能。同源下一个标签页修改 localStorage,其他标签页会触发 `storage` 事件,可用于简单的跨标签页通信。

**Q:为什么 cookie 有容量限制且不宜存大数据?**
A:cookie 会随每个同域请求自动发送到服务器,过大浪费带宽、拖慢请求,所以限制约 4KB,只适合存少量必要标识。

**Q:sessionStorage 在什么情况下会保留/清除?**
A:同一标签页刷新、跳转同源页面会保留;关闭标签页或打开新标签页则不共享、清除。它绑定到具体的标签页会话。

## 一句话背诵版

**cookie(4KB、可设过期、随请求自动携带、身份认证)、localStorage(5-10MB、永久、本地、用户设置)、sessionStorage(会话级、关标签清除)、IndexedDB(大容量、异步事务、离线数据);token 优先 HttpOnly+Secure+SameSite cookie。**
