# V2 · 组件通信方式

> 热度:🔥🔥🔥

## 一句话考点

Vue 组件通信:父子用 **props / $emit**,跨层级用 **provide/inject**,任意组件用 **事件总线(Vue2)/ mitt(Vue3)** 或 **Pinia/Vuex**,还有 `$refs`、`$attrs`、`v-model` 等。

## 原理精讲

### 通信方式一览

| 场景 | Vue2 | Vue3 |
|------|------|------|
| 父 → 子 | props | props |
| 子 → 父 | `$emit` 触发自定义事件 | `emit`(defineEmits) |
| 双向绑定 | `v-model` / `.sync` | `v-model`(可多个) |
| 跨层级 | `provide / inject` | `provide / inject` |
| 父访问子 | `$refs` | `ref` + defineExpose |
| 透传属性 | `$attrs` / `$listeners` | `$attrs`(合并了 listeners) |
| 任意组件 | EventBus(new Vue) | mitt 库 |
| 全局状态 | Vuex | Pinia |

### v-model 原理

`v-model` 是语法糖:
- Vue2:`value` prop + `input` 事件;
- Vue3:`modelValue` prop + `update:modelValue` 事件,且支持多个 `v-model:xxx`。

## 代码示例

### 父子通信(Vue3 `<script setup>`)

```vue
<!-- 子组件 Child.vue -->
<script setup>
const props = defineProps(['title']);          // 父 → 子
const emit = defineEmits(['submit']);           // 子 → 父
function onClick() {
  emit('submit', { id: 1 });                    // 触发事件传参
}
</script>
<template>
  <div>{{ title }}<button @click="onClick">提交</button></div>
</template>
```

```vue
<!-- 父组件 -->
<template>
  <Child :title="标题" @submit="handleSubmit" />
</template>
```

### 跨层级 provide / inject

```js
// 祖先组件提供
import { provide, ref } from 'vue';
const theme = ref('dark');
provide('theme', theme);      // 提供响应式数据

// 后代组件注入(无论隔多少层)
import { inject } from 'vue';
const theme = inject('theme', 'light');  // 第二参数为默认值
```

### 自定义 v-model(Vue3)

```vue
<!-- 子组件 -->
<script setup>
defineProps(['modelValue']);
const emit = defineEmits(['update:modelValue']);
</script>
<template>
  <input :value="modelValue" @input="emit('update:modelValue', $event.target.value)" />
</template>

<!-- 父组件 -->
<Child v-model="text" />
```

## 高频追问

**Q:Vue 有哪些组件通信方式?**
A:props(父子)、$emit(子父)、v-model(双向)、provide/inject(跨层级)、$refs(父调子)、$attrs(透传)、EventBus/mitt(任意)、Vuex/Pinia(全局)。

**Q:provide/inject 有什么注意点?**
A:默认不是响应式的,要传 ref/reactive 才响应式;它建立祖先-后代的隐式依赖,过度使用会降低可维护性,适合主题、i18n、表单上下文等场景。

**Q:v-model 的原理?Vue2 和 Vue3 区别?**
A:v-model 是 prop + 事件的语法糖。Vue2 默认 `value` + `input`,一个组件只能一个 v-model;Vue3 默认 `modelValue` + `update:modelValue`,支持多个 `v-model:xxx`,`.sync` 被并入 v-model。

**Q:Vue3 为什么移除了 EventBus?**
A:Vue3 移除了 `$on/$off/$emit` 实例方法,无法直接 new Vue 做总线。改用第三方库 mitt 实现发布订阅,或用 Pinia 管理跨组件状态。

**Q:$attrs 是什么?**
A:未被 props 声明接收的父级传入属性(class、style、普通属性、事件),可透传给内部元素。Vue3 中 `$attrs` 合并了原 `$listeners`,配合 `inheritAttrs: false` 可自定义透传位置。

## 一句话背诵版

**父子 props/$emit、双向 v-model(Vue3 = modelValue + update:modelValue 可多个)、跨层级 provide/inject(传 ref 才响应)、父调子 $refs、透传 $attrs、任意组件 mitt/EventBus、全局 Pinia/Vuex。**
