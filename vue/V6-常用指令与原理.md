# V6 · 常用指令与原理

> 热度:🔥🔥

## 一句话考点

指令是带 `v-` 前缀的特殊属性;高频区分 `v-if`(销毁重建,惰性)vs `v-show`(切换 display);`v-for` 要配 key;`v-model` 是双向绑定语法糖。

## 原理精讲

### 1. 常用内置指令

| 指令 | 作用 |
|------|------|
| `v-if / v-else-if / v-else` | 条件渲染(增删 DOM) |
| `v-show` | 条件显示(切换 `display`) |
| `v-for` | 列表渲染 |
| `v-bind`(`:`) | 绑定属性 |
| `v-on`(`@`) | 绑定事件 |
| `v-model` | 双向绑定 |
| `v-html` | 渲染 HTML(注意 XSS) |
| `v-once` | 只渲染一次(静态内容优化) |
| `v-memo`(Vue3) | 缓存子树,依赖不变跳过更新 |

### 2. v-if vs v-show

| | v-if | v-show |
|--|------|--------|
| 原理 | 增删 DOM 节点 | 切换 `display: none` |
| 初始开销 | 惰性(false 时不渲染) | 总会渲染 |
| 切换开销 | 大(销毁/重建 + 生命周期) | 小(只改样式) |
| 适用 | 很少切换、条件初始为假 | 频繁切换 |

### 3. v-for 与 v-if 优先级

- **Vue2**:v-for 优先级高于 v-if(同元素上 v-for 先执行,每次循环都判断 v-if,性能差);
- **Vue3**:v-if 优先级更高(会报警告);
- **最佳实践**:不要在同一元素同时用,用 computed 过滤列表,或把 v-if 提到外层。

### 4. v-model 原理

语法糖 = 属性绑定 + 事件监听(见 V2)。表单元素上默认对应 value+input(文本)、checked+change(勾选)等。

### 5. 自定义指令

用于直接操作 DOM 的可复用逻辑(如聚焦、懒加载、权限),有 `mounted`、`updated` 等钩子。

## 代码示例

```vue
<!-- v-if 惰性:false 时根本不渲染 -->
<HeavyComp v-if="show" />
<!-- v-show:始终渲染,只切 display -->
<div v-show="show">频繁切换用我</div>

<!-- v-for 配 key,避免同时用 v-if -->
<!-- ❌ 不要:<li v-for="u in users" v-if="u.active"> -->
<li v-for="u in activeUsers" :key="u.id">{{ u.name }}</li>

<!-- 用 computed 过滤 -->
<script setup>
const activeUsers = computed(() => users.value.filter(u => u.active));
</script>
```

### 自定义指令(v-focus)

```js
// 全局注册
app.directive('focus', {
  mounted(el) {
    el.focus();          // 元素挂载后自动聚焦
  },
});
// 使用:<input v-focus />
```

## 高频追问

**Q:v-if 和 v-show 区别?怎么选?**
A:v-if 通过增删 DOM 实现条件渲染,有更高切换开销但初始为假时不渲染(惰性);v-show 通过 display 切换,初始总渲染但切换廉价。频繁切换用 v-show,很少切换或初始不显示用 v-if。

**Q:为什么不能 v-for 和 v-if 一起用?**
A:Vue2 中 v-for 优先级更高,每次循环都会执行 v-if 判断,浪费性能;Vue3 中 v-if 优先级更高,此时拿不到 v-for 的循环变量会出错。应改用 computed 预过滤或拆分层级。

**Q:v-html 有什么风险?**
A:直接渲染 HTML 字符串,若内容来自用户输入会有 XSS 注入风险。只对可信内容使用,必要时先做净化(如 DOMPurify)。

**Q:v-once 和 v-memo 作用?**
A:v-once 让元素/组件只渲染一次,后续更新跳过,适合永不变化的静态内容;v-memo(Vue3)传入依赖数组,依赖不变时跳过该子树的更新,用于优化大列表。

**Q:自定义指令用在什么场景?**
A:需要直接、复用性地操作 DOM 的场景:自动聚焦、图片懒加载、无限滚动、权限控制(v-permission)、点击外部关闭等。

## 一句话背诵版

**v-if 增删 DOM(惰性、切换贵)、v-show 切 display(总渲染、切换廉价,频繁切换用它);v-for 必配 key、别和 v-if 同元素(用 computed 过滤);v-model 是属性+事件语法糖;v-html 防 XSS;自定义指令封装 DOM 操作。**
