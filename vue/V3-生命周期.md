# V3 · 生命周期

> 热度:🔥🔥

## 一句话考点

Vue 生命周期分**创建、挂载、更新、销毁**四阶段;Vue3 Composition API 里除 `beforeCreate/created` 被 `setup` 取代外,其余钩子加 `on` 前缀(如 `onMounted`)。

## 原理精讲

### 1. Vue2 生命周期钩子

| 阶段 | 钩子 | 说明 |
|------|------|------|
| 创建 | `beforeCreate` | 实例初始化,data/methods 还不可用 |
| | `created` | data 可用,**未挂载 DOM**(常用:请求数据) |
| 挂载 | `beforeMount` | 挂载前,虚拟 DOM 已创建 |
| | `mounted` | DOM 已挂载(常用:操作 DOM、初始化第三方库) |
| 更新 | `beforeUpdate` | 数据变化、DOM 更新前 |
| | `updated` | DOM 更新后 |
| 销毁 | `beforeDestroy` | 销毁前(常用:清定时器、解绑事件) |
| | `destroyed` | 销毁后 |

> Vue3 更名:`beforeDestroy → beforeUnmount`,`destroyed → unmounted`。

### 2. Vue3 Composition API 对应

| 选项式 | 组合式(setup 内) |
|--------|-----------------|
| beforeCreate / created | 直接写在 `setup()` 里 |
| beforeMount | `onBeforeMount` |
| mounted | `onMounted` |
| beforeUpdate | `onBeforeUpdate` |
| updated | `onUpdated` |
| beforeUnmount | `onBeforeUnmount` |
| unmounted | `onUnmounted` |

### 3. keep-alive 专属钩子

被 `<keep-alive>` 缓存的组件有额外钩子:
- `activated`:被激活(进入)时;
- `deactivated`:被缓存(离开)时。

## 代码示例

### Vue3 组合式

```vue
<script setup>
import { onMounted, onUnmounted, ref } from 'vue';

const data = ref(null);

// setup 内直接执行 = created 时机
console.log('相当于 created');

onMounted(async () => {
  data.value = await fetchData();     // DOM 已挂载,可操作 DOM
  console.log('mounted');
});

onUnmounted(() => {
  clearInterval(timer);               // 清理副作用
  console.log('unmounted');
});
</script>
```

### 父子组件生命周期顺序(高频)

```
挂载:
父 beforeCreate → 父 created → 父 beforeMount
→ 子 beforeCreate → 子 created → 子 beforeMount → 子 mounted
→ 父 mounted

更新:
父 beforeUpdate → 子 beforeUpdate → 子 updated → 父 updated

销毁:
父 beforeUnmount → 子 beforeUnmount → 子 unmounted → 父 unmounted
```

## 高频追问

**Q:请求数据放 created 还是 mounted?**
A:都可以。`created` 更早(能更快发起请求,SSR 也会执行);若请求结果需要操作 DOM,则放 `mounted`。多数场景放 created/setup。

**Q:created 和 mounted 区别?**
A:created 时 data/methods 已就绪但 DOM 未生成、`$el` 不可用;mounted 时真实 DOM 已挂载,可访问 `$refs`、操作 DOM、初始化图表等。

**Q:父子组件挂载顺序?**
A:父 beforeMount 后先完成子组件的完整挂载(子 mounted),父组件才 mounted。即"父创建→子创建挂载→父挂载",子的 mounted 早于父的 mounted。

**Q:Vue3 setup 对应哪个生命周期?**
A:setup 在 beforeCreate 之前执行(此时组件实例还未完全创建),它取代了 beforeCreate 和 created,所以这两个钩子的逻辑直接写在 setup 里。

**Q:keep-alive 组件的生命周期?**
A:首次进入正常触发 mounted;之后切换不再走 mounted/unmounted,而是走 `activated`(激活)和 `deactivated`(缓存),用于恢复/暂停状态。

## 一句话背诵版

**四阶段:创建(beforeCreate/created,created 可用 data 常用于请求)、挂载(beforeMount/mounted,mounted 可操作 DOM)、更新(beforeUpdate/updated)、销毁(beforeUnmount/unmounted 清理);Vue3 钩子加 on 前缀、setup 取代 created;父子挂载"父创建→子挂载→父挂载"。**
