# V1 · 响应式原理(Vue2 / Vue3)

> 热度:🔥🔥🔥

## 一句话考点

Vue2 用 `Object.defineProperty` 劫持属性的 getter/setter 实现响应式;Vue3 改用 **Proxy** 代理整个对象,解决了 Vue2 无法监听新增属性、数组下标、需要递归遍历等缺陷。

## 原理精讲

### 1. Vue2:Object.defineProperty

初始化时递归遍历 data 的每个属性,用 `defineProperty` 定义 getter/setter:
- **getter**:依赖收集(把当前 watcher 加入 dep);
- **setter**:派发更新(通知 dep 里的 watcher 更新)。

**缺陷**:
1. 无法检测**对象属性的新增/删除**(需 `Vue.set` / `Vue.delete`);
2. 无法检测**数组通过下标修改**或修改 length(Vue2 重写了 push/pop 等 7 个方法来 hack);
3. 初始化时需**递归遍历**所有属性,性能开销大(即使从未用到)。

### 2. Vue3:Proxy

用 `Proxy` 代理整个对象,拦截 13 种操作(get/set/deleteProperty/has 等):
- 天然支持属性新增/删除、数组下标修改;
- **懒代理**:访问到嵌套对象时才递归代理(reactive 的 get 中按需处理),性能更好;
- 配合 `Reflect` 完成默认行为。

### 3. 依赖收集机制

- Vue2:Dep(依赖管理)+ Watcher(观察者),getter 收集、setter 通知;
- Vue3:`track`(收集)+ `trigger`(触发),用 WeakMap 存储 `对象→属性→effect` 的依赖关系,`effect` 是副作用函数。

## 代码示例

### Vue3 reactive 简版原理

```js
function reactive(obj) {
  return new Proxy(obj, {
    get(target, key, receiver) {
      track(target, key);                          // 依赖收集
      const res = Reflect.get(target, key, receiver);
      // 懒代理:嵌套对象访问时才代理
      return typeof res === 'object' && res !== null ? reactive(res) : res;
    },
    set(target, key, value, receiver) {
      const res = Reflect.set(target, key, value, receiver);
      trigger(target, key);                        // 派发更新
      return res;
    },
    deleteProperty(target, key) {
      const res = Reflect.deleteProperty(target, key);
      trigger(target, key);                        // 删除也能触发
      return res;
    },
  });
}
```

### Vue2 的坑与解法

```js
// ❌ Vue2 中这些不触发更新
this.obj.newProp = 1;        // 新增属性
this.arr[0] = 2;             // 下标修改
this.arr.length = 0;         // 修改长度

// ✅ 解法
this.$set(this.obj, 'newProp', 1);
this.arr.splice(0, 1, 2);    // 用被重写的数组方法
```

## 高频追问

**Q:Vue2 和 Vue3 响应式原理区别?**
A:Vue2 用 Object.defineProperty 劫持每个属性的 getter/setter,需递归遍历、无法监听属性增删和数组下标;Vue3 用 Proxy 代理整个对象,支持增删/数组下标、懒递归代理、性能更好。

**Q:为什么 Vue3 用 Proxy 替代 defineProperty?**
A:defineProperty 只能劫持已存在的属性,新增/删除、数组下标改动监听不到,且要初始化时深度遍历。Proxy 拦截整个对象的所有操作,天然解决这些问题且可懒代理。

**Q:Vue2 中为什么数组修改经常不生效?**
A:defineProperty 不能高效劫持数组下标和 length。Vue2 通过重写 push/pop/shift/unshift/splice/sort/reverse 七个方法实现响应式,直接用下标赋值或改 length 不会触发更新,要用这些方法或 `$set`。

**Q:Vue3 的 ref 和 reactive 区别?**
A:reactive 用于对象/数组(返回 Proxy);ref 用于任意值(包括原始类型),内部用对象 `{ value }` 包裹再 reactive 化,访问要 `.value`(模板中自动解包)。

**Q:响应式的依赖收集在什么时候发生?**
A:在渲染/计算读取(触发 getter)时收集当前的副作用(watcher/effect);数据被修改(触发 setter)时通知这些副作用重新执行。即"读时收集,写时触发"。

## 一句话背诵版

**Vue2 用 defineProperty 劫持 getter/setter(读时依赖收集、写时派发更新),缺陷是监听不到属性增删和数组下标、需递归遍历;Vue3 用 Proxy 代理整个对象,支持增删/下标、懒递归、性能更好,配合 track/trigger + WeakMap。**
