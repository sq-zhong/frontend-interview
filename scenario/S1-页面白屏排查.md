# S1 · 页面白屏排查

> 场景排错专项 · 实战模板:现象 → 排查路径 → 常见根因 → 解决 → 举一反三

## 现象

用户打开页面一片空白,或某次发布后线上白屏、本地却正常。这是最高频的线上事故。

## 排查路径(从外到内逐层缩小)

```
① 打开控制台看有没有报错 / 白屏是"真空白"还是"有骨架无内容"
② Network 面板:HTML / JS / CSS 是否 200 加载成功?
   - HTML 404/500 → 部署或路由问题
   - JS 加载失败(404/跨域/CORS)→ 资源路径或 CDN 问题
   - JS 加载成功但没执行 → 看 Console 报错
③ Console 有红色报错吗?
   - 语法/运行时错误导致 JS 中断 → 定位报错栈
   - chunk 加载失败(ChunkLoadError)→ 缓存/发布问题
④ Elements 看 #app / #root 是否为空
   - 空 → JS 没渲染(框架报错、路由没匹配)
   - 有内容但不可见 → CSS 问题(见 S8)
⑤ 有无 sourcemap 定位到具体代码行
```

## 常见根因(分类)

**1. 资源加载失败**
- 打包路径错误(`publicPath` / `base` 配置不对,资源 404);
- CDN 挂了或跨域;
- 发布后旧 HTML 引用了新 hash 的 JS(或反之),chunk 404 → `ChunkLoadError`。

**2. JS 运行时报错中断**
- 某个同步错误抛出,导致整个 bundle 执行中断,页面无渲染;
- 依赖了某个环境不存在的 API(如低版本浏览器不支持的语法/API,没做 polyfill)。

**3. 框架层问题**
- 路由没匹配到(如 history 模式服务端没配置 fallback,刷新子路由 404);
- 根组件渲染报错(React 错误边界外的错误会卸载整棵树 → 白屏);
- 接口挂了且没做兜底,数据 undefined 导致渲染崩溃。

**4. 兼容性**
- 用了 ES6+ 语法但目标浏览器不支持,且未经 Babel 转译(如给老安卓/iOS);
- CSS/JS 特性不兼容。

## 解决方案

| 根因 | 解决 |
|------|------|
| 资源路径 404 | 检查 `publicPath`/`base`、CDN、部署目录 |
| ChunkLoadError | HTML 不强缓存(no-cache)、JS 用 contenthash;监听该错误自动刷新 |
| 运行时报错 | 修复报错;关键渲染包 try/catch;React 用错误边界 |
| history 路由刷新 404 | 服务端配置 fallback 到 index.html |
| 兼容性 | 配置 Babel + core-js polyfill、browserslist |
| 接口兜底 | 数据加载失败展示兜底 UI / 骨架屏 |

## 举一反三(面试如何答)

**Q:线上白屏怎么排查?**
> 我会分层排查:先看 Console 有无报错、Network 资源是否加载成功;若 JS 404 多半是发布/路径问题,若 JS 报错就看错误栈;再看 root 节点是否渲染,空的话查框架/路由/接口。**最常见的两类是:发布导致的 chunk 加载失败(HTML 和 JS 缓存不一致),以及某个未捕获的运行时错误让整个 bundle 中断。**

**关键补充**:
- **要有监控**:接白屏检测(如监控 root 节点在 N 秒后是否仍为空)+ 前端错误上报(Sentry),否则你根本不知道有多少用户白屏;
- **要有兜底**:错误边界、chunk 加载失败自动重试/刷新、接口失败展示重试按钮;
- **发布策略**:HTML no-cache + 静态资源长缓存 + 灰度发布,避免新旧资源不匹配。

## 一句话总结

**白屏排查四层:Console 报错 → Network 资源 → root 是否渲染 → CSS 是否隐藏;高频根因是发布导致的 ChunkLoadError(缓存不一致)和运行时错误中断 bundle;根治靠错误监控 + 错误边界兜底 + HTML no-cache 发布策略。**
