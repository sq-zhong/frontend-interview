# S3 · 内存泄漏定位

> 场景排错专项 · 现象 → 排查路径 → 常见根因 → 解决 → 举一反三

## 现象

页面用久了越来越卡、最终崩溃刷新;单页应用反复切换路由后内存持续上涨不回落;长时间运行的页面(大屏、后台)内存爆掉。

## 排查路径(Memory 面板 + 曲线)

```
① Performance 面板录制:勾选 Memory,看 JS Heap 曲线
   - 正常:锯齿状(涨了又被 GC 回收)
   - 泄漏:整体阶梯式上涨,GC 后也不回落
② Memory 面板 → Heap Snapshot(堆快照)对比法:
   - 操作前拍一张 → 反复执行可疑操作(如切路由 N 次)→ 手动 GC → 再拍一张
   - 用 "Comparison" 视图看哪些对象数量持续增长(Delta 为正)
③ 点进增长的对象,看 Retainers(引用链)——谁在引用它、导致无法回收
④ 定位到具体代码:通常是某个未解绑的监听/定时器/闭包
```

## 常见根因(分类)

**1. 未清理的定时器 / 监听器**
- `setInterval` 没 clear,回调一直引用外部变量;
- `addEventListener` 没 `removeEventListener`(尤其绑在 window/document 上的);
- 组件卸载了但订阅还在(EventBus、WebSocket、IntersectionObserver)。

**2. 闭包持有大对象**
- 闭包长期引用大数组/大对象,始终无法回收(见 J3/J11)。

**3. 脱离 DOM 的引用**
- DOM 从页面移除了,但 JS 变量(或数组/Map)还引用着它 → 游离 DOM 泄漏。

**4. 框架层**
- React:useEffect 没返回清理函数;闭包捕获旧 state;缓存无限增长;
- Vue:全局事件/第三方实例在组件销毁时没销毁;`keep-alive` 缓存过多。

**5. 全局缓存无上限**
- 用 Map/对象做缓存却从不清理,只增不减。

## 解决方案

| 根因 | 解决 |
|------|------|
| 定时器 | 组件卸载时 clearInterval/clearTimeout |
| 事件监听 | 卸载时 removeEventListener(注意要同一函数引用) |
| 订阅/连接 | 卸载时取消订阅、关闭 WebSocket、observer.disconnect() |
| 游离 DOM | 移除 DOM 后把引用置 null |
| 缓存 | 用 WeakMap(键弱引用自动回收)或设 LRU 上限 |
| React | useEffect 返回清理函数;依赖写全 |
| Vue | onUnmounted 里销毁定时器/监听/实例 |

## 代码示例

```jsx
// ❌ 泄漏:监听没解绑,定时器没清
useEffect(() => {
  const timer = setInterval(fetchData, 1000);
  window.addEventListener('resize', onResize);
}, []);

// ✅ 返回清理函数
useEffect(() => {
  const timer = setInterval(fetchData, 1000);
  window.addEventListener('resize', onResize);
  return () => {
    clearInterval(timer);
    window.removeEventListener('resize', onResize);   // 必须同一函数引用
  };
}, []);
```

## 举一反三(面试如何答)

**Q:怎么排查内存泄漏?**
> 先用 Performance 录制看 JS Heap 曲线是不是 GC 后仍持续上涨。确认泄漏后用 Memory 面板的**堆快照对比法**:操作前后各拍一张快照,对比找出持续增长的对象,再看它的 Retainers 引用链定位是谁没释放。**最常见的根因是定时器、事件监听、订阅在组件卸载时没清理,以及闭包/缓存长期持有大对象。**

**关键补充**:
- 强调**快照对比 + Retainers 引用链**是核心手法,不是靠猜;
- 提 **WeakMap/WeakRef** 做缓存能自动回收,体现深度;
- 结合框架:"React 就是 useEffect 忘了返回清理函数,Vue 就是 onUnmounted 没销毁"。

## 一句话总结

**泄漏排查:Performance 看 Heap 曲线是否只涨不落 → Memory 堆快照对比找增长对象 → Retainers 引用链定位;高频根因是定时器/监听/订阅未在卸载时清理、闭包和缓存持有大对象、游离 DOM;根治靠清理函数、WeakMap 缓存、置 null。**
