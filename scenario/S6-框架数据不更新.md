# S6 · 框架数据不更新(响应式失效)

> 场景排错专项 · 现象 → 排查路径 → 常见根因 → 解决 → 举一反三

## 现象

改了数据但视图没变;React setState 后拿到的还是旧值;Vue 里数组/对象改了页面不刷新;明明数据对了,组件就是不重渲染。

## 排查路径

```
① 数据到底变了没?console.log / DevTools 看真实值
   - 数据没变 → 逻辑 bug(改错了对象、异步没回来)
   - 数据变了但视图没动 → 响应式/渲染问题
② React:
   - 是不是直接改了 state(mutation)而非返回新引用?
   - 是不是闭包拿了旧 state?
   - key 是否稳定导致复用错误(见 R6)?
③ Vue:
   - Vue2:是不是数组下标/新增属性/删除属性?(defineProperty 限制,见 V1)
   - Vue3:是不是解构 reactive 丢了响应式?
④ 依赖是否正确(useEffect 依赖、computed 依赖)?
```

## 常见根因(分类)

### React

**1. 直接修改 state(未产生新引用)**
```jsx
// ❌ 直接 mutate,引用没变,React 认为没更新
const [list, setList] = useState([]);
list.push(item); setList(list);          // 不触发渲染

// ✅ 返回新引用
setList([...list, item]);
setUser({ ...user, name: 'new' });
```

**2. 闭包捕获旧 state**
```jsx
// ❌ 定时器里的 count 永远是初始值(见 R5)
useEffect(() => {
  setInterval(() => setCount(count + 1), 1000);  // count 停留在 0
}, []);
// ✅ 函数式更新
setInterval(() => setCount(c => c + 1), 1000);
```

**3. setState 是异步批处理**,同一事件里连续读拿不到最新值(见 R4)。

### Vue

**1. Vue2 响应式限制(见 V1)**
```js
this.arr[0] = 1;          // ❌ 下标修改不触发
this.obj.newKey = 1;      // ❌ 新增属性不触发
// ✅
this.$set(this.arr, 0, 1);
this.arr.splice(0, 1, 1);
```

**2. Vue3 解构丢响应式**
```js
const state = reactive({ count: 0 });
let { count } = state;    // ❌ 解构后 count 是普通值
// ✅ 用 toRefs
const { count } = toRefs(state);
```

**3. computed 依赖没被追踪**(依赖了非响应式数据,或异步赋值)。

## 解决方案速查

| 框架 | 问题 | 解决 |
|------|------|------|
| React | 直接 mutate | 返回新引用(展开/immer) |
| React | 闭包旧值 | 函数式更新 `setX(prev=>...)` |
| React | 依赖漏写 | 补全 useEffect/useMemo 依赖 |
| Vue2 | 数组下标/新增属性 | `$set` / splice |
| Vue3 | 解构丢响应式 | toRefs / storeToRefs |
| Vue | computed 不更新 | 确保依赖是响应式的 |

## 举一反三(面试如何答)

**Q:React setState 了但视图没更新,可能是什么原因?**
> 最常见三种:**① 直接修改了 state 而没有返回新引用**(React 用 Object.is 浅比较,引用没变就不重渲染),要用展开运算符或 immer 产生新对象;**② 闭包捕获了旧的 state**,比如定时器里用了 count,要改成函数式更新 `setCount(c => c+1)`;③ setState 批处理,同一事件里连续调用基于的是同一次渲染的旧值。

**Vue 版本**:
> Vue2 是 `Object.defineProperty` 的限制——数组下标改、对象新增/删除属性不触发响应式,要用 `$set` 或数组变异方法;Vue3 用 Proxy 解决了这些,但**解构 reactive 对象会丢失响应式**,要用 toRefs。

**关键**:这题直接连着 **R4/R5(Hooks)、V1(响应式原理)** 的八股——能把"为什么"讲清楚(浅比较、defineProperty vs Proxy)就是深度。

## 一句话总结

**数据不更新先确认数据真变了没;React 三大坑:直接 mutate 未换引用(浅比较不更新)、闭包拿旧 state(用函数式更新)、批处理;Vue2 坑是数组下标/属性增删(用 $set/splice)、Vue3 坑是解构丢响应式(用 toRefs);本质是 R4/R5/V1 的实战。**
