# R3 · 生命周期与函数组件

> 热度:🔥🔥

## 一句话考点

类组件生命周期分**挂载、更新、卸载**三阶段;函数组件用 **Hooks** 替代,`useEffect` 统一模拟了 `componentDidMount / DidUpdate / WillUnmount`。

## 原理精讲

### 1. 类组件生命周期(三阶段)

**挂载**:
- `constructor` → `getDerivedStateFromProps` → `render` → `componentDidMount`(常用:请求数据、订阅)

**更新**(props/state 变化):
- `getDerivedStateFromProps` → `shouldComponentUpdate`(性能优化,返回 false 跳过渲染)→ `render` → `getSnapshotBeforeUpdate` → `componentDidUpdate`

**卸载**:
- `componentWillUnmount`(清理:取消订阅、清定时器)

> 已废弃(unsafe):`componentWillMount`、`componentWillReceiveProps`、`componentWillUpdate`。

### 2. 函数组件 + Hooks 对应关系

| 类组件 | 函数组件 |
|--------|---------|
| `componentDidMount` | `useEffect(fn, [])` |
| `componentDidUpdate` | `useEffect(fn, [deps])` |
| `componentWillUnmount` | `useEffect` 的 return 清理函数 |
| `shouldComponentUpdate` | `React.memo` |
| `this.state` / `setState` | `useState` |

### 3. 函数组件 vs 类组件

- 函数组件:更简洁、无 this 困扰、逻辑复用靠自定义 Hook,是**官方推荐**;
- 类组件:有完整生命周期钩子,老项目常见;
- React 16.8+ Hooks 后,新项目基本全用函数组件。

## 代码示例

### 类组件

```jsx
class Timer extends React.Component {
  state = { seconds: 0 };

  componentDidMount() {          // 挂载后启动定时器
    this.timer = setInterval(() => {
      this.setState(s => ({ seconds: s.seconds + 1 }));
    }, 1000);
  }
  componentWillUnmount() {       // 卸载前清理
    clearInterval(this.timer);
  }
  render() {
    return <div>{this.state.seconds}s</div>;
  }
}
```

### 等价函数组件

```jsx
function Timer() {
  const [seconds, setSeconds] = useState(0);

  useEffect(() => {
    const timer = setInterval(() => {
      setSeconds(s => s + 1);
    }, 1000);
    return () => clearInterval(timer);   // 清理 = componentWillUnmount
  }, []);                                 // [] = 只在挂载时执行

  return <div>{seconds}s</div>;
}
```

## 高频追问

**Q:请求数据放在哪个生命周期?为什么?**
A:类组件放 `componentDidMount`,函数组件放 `useEffect(fn, [])`。因为此时 DOM 已挂载,且不会阻塞首次渲染;放 render/constructor 会有副作用或阻塞问题。

**Q:`componentWillMount` 为什么被废弃?**
A:它在 Fiber 架构的异步渲染下可能被多次调用,容易产生副作用 bug(如重复请求);相关 unsafe 生命周期都因异步渲染的不确定性被废弃。

**Q:`useEffect` 如何模拟三个生命周期?**
A:依赖数组 `[]` → 只在挂载执行(DidMount);`[deps]` → deps 变化时执行(DidUpdate);return 的函数 → 卸载或下次执行前调用(WillUnmount)。

**Q:函数组件相比类组件的优势?**
A:更简洁、无 this 绑定问题、逻辑复用更优雅(自定义 Hook 替代 HOC/render props)、更贴合 React 未来方向(并发特性)。

## 一句话背诵版

**类组件三阶段:挂载(constructor→render→DidMount)、更新(shouldUpdate→render→DidUpdate)、卸载(WillUnmount);函数组件用 useEffect 统一:`[]`=DidMount、`[deps]`=DidUpdate、return=WillUnmount;数据请求放 DidMount / useEffect([])。**
