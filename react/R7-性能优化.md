# R7 · 性能优化(memo / useMemo / useCallback)

> 热度:🔥🔥🔥

## 一句话考点

React 默认父组件重渲染会导致子组件重渲染;用 `React.memo` 缓存组件、`useMemo` 缓存计算、`useCallback` 缓存函数引用,配合稳定的 props 避免不必要的重渲染。

## 原理精讲

### 1. React 何时重渲染

- 自身 state 变化;
- 父组件重渲染(默认连带子组件一起,无论 props 是否变);
- 消费的 Context value 变化。

### 2. 三个优化 API

| API | 缓存什么 | 作用 |
|-----|---------|------|
| `React.memo(Comp)` | 组件 | props 浅比较相同则跳过重渲染 |
| `useMemo(fn, deps)` | 计算结果(值) | 依赖不变则不重新计算 |
| `useCallback(fn, deps)` | 函数引用 | 依赖不变则返回同一函数引用 |

### 3. 三者如何配合

`React.memo` 靠 props 浅比较。但每次渲染时,内联函数和对象都是**新引用**,会让 memo 失效。因此:
- 传给 memo 组件的**函数** props 用 `useCallback` 稳定引用;
- 传给 memo 组件的**对象/数组** props 用 `useMemo` 稳定引用;
- 昂贵计算用 `useMemo` 避免重复计算。

### 4. 不要过度优化

memo/useMemo/useCallback 本身有比较和缓存成本。只在**确有性能问题**(大列表、昂贵计算、频繁渲染)时用,滥用反而增加复杂度和开销。

## 代码示例

### memo + useCallback 配合

```jsx
const Child = React.memo(function Child({ onClick }) {
  console.log('Child render');
  return <button onClick={onClick}>点击</button>;
});

function Parent() {
  const [count, setCount] = useState(0);

  // ❌ 每次渲染都是新函数,memo 失效,Child 每次都重渲染
  // const handleClick = () => console.log('click');

  // ✅ useCallback 稳定引用,count 变化时 Child 不再重渲染
  const handleClick = useCallback(() => console.log('click'), []);

  return (
    <>
      <button onClick={() => setCount(c => c + 1)}>{count}</button>
      <Child onClick={handleClick} />
    </>
  );
}
```

### useMemo 缓存昂贵计算

```jsx
function List({ items, keyword }) {
  // 仅当 items 或 keyword 变化时才重新过滤
  const filtered = useMemo(
    () => items.filter(i => i.name.includes(keyword)),
    [items, keyword]
  );
  return filtered.map(i => <li key={i.id}>{i.name}</li>);
}
```

## 高频追问

**Q:React.memo、useMemo、useCallback 分别缓存什么?**
A:memo 缓存组件(props 浅比较跳过重渲染);useMemo 缓存计算结果(值);useCallback 缓存函数引用。`useCallback(fn, d)` 等价 `useMemo(() => fn, d)`。

**Q:为什么给 memo 组件传函数要配 useCallback?**
A:memo 做 props 浅比较。内联函数每次渲染都是新引用,导致浅比较不等、memo 失效。useCallback 让函数引用稳定,memo 才能生效。

**Q:是不是所有函数都该用 useCallback 包?**
A:不是。只有当函数作为 props 传给 memo 化的子组件,或作为其他 Hook 的依赖时才需要。无意义的包裹只增加开销。

**Q:如何定位不必要的重渲染?**
A:React DevTools 的 Profiler(可高亮渲染)、`why-did-you-render` 库,或在组件里打 log 观察渲染次数。

**Q:除了这三个 API,还有哪些性能优化手段?**
A:列表虚拟化(react-window)、代码分割(lazy + Suspense)、合理拆分组件缩小渲染范围、Context 拆分、避免在 render 中创建新对象、useTransition 降低更新优先级。

## 一句话背诵版

**父渲染默认连带子渲染;React.memo 缓存组件(props 浅比较)、useMemo 缓存值、useCallback 缓存函数;给 memo 组件传的函数/对象 props 要用 useCallback/useMemo 稳定引用否则失效;别过度优化,有性能问题再上。**
