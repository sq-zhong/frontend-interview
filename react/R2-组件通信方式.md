# R2 · 组件通信方式

> 热度:🔥🔥🔥

## 一句话考点

React 数据流是**单向**的:父传子用 **props**,子传父用**回调函数**,跨层级用 **Context**,兄弟/全局用**状态管理**(Redux/Zustand)或状态提升。

## 原理精讲

### 通信场景与方案

| 场景 | 方案 |
|------|------|
| 父 → 子 | props |
| 子 → 父 | 父传回调函数,子调用并传参 |
| 兄弟组件 | 状态提升到共同父组件 |
| 跨多层级 | Context(避免 props 层层透传) |
| 任意组件/全局 | 状态管理库(Redux、Zustand、Jotai) |
| 非父子任意通信 | 发布订阅(EventEmitter) |
| 父调子的方法 | ref + useImperativeHandle |

### 单向数据流

数据从父流向子,子不能直接修改 props(只读)。要改变要通过回调通知父组件更新 state,再重新渲染。好处:数据流向清晰、可预测、易调试。

## 代码示例

### 父子 & 子父

```jsx
// 父组件
function Parent() {
  const [count, setCount] = useState(0);
  return (
    <Child
      count={count}                         // 父 → 子:props
      onIncrement={() => setCount(c => c + 1)} // 子 → 父:回调
    />
  );
}
function Child({ count, onIncrement }) {
  return <button onClick={onIncrement}>{count}</button>;
}
```

### Context 跨层级

```jsx
const ThemeContext = createContext('light');

function App() {
  return (
    <ThemeContext.Provider value="dark">
      <Toolbar />
    </ThemeContext.Provider>
  );
}
function Toolbar() {
  return <ThemedButton />;   // 中间层无需透传
}
function ThemedButton() {
  const theme = useContext(ThemeContext);  // 直接消费
  return <button className={theme}>按钮</button>;
}
```

### ref 调用子组件方法

```jsx
const Child = forwardRef((props, ref) => {
  useImperativeHandle(ref, () => ({
    focus: () => inputRef.current.focus(),
  }));
  const inputRef = useRef();
  return <input ref={inputRef} />;
});

function Parent() {
  const childRef = useRef();
  return (
    <>
      <Child ref={childRef} />
      <button onClick={() => childRef.current.focus()}>聚焦</button>
    </>
  );
}
```

## 高频追问

**Q:React 有哪些组件通信方式?**
A:props(父子)、回调(子父)、状态提升(兄弟)、Context(跨层级)、状态管理库(全局)、ref+useImperativeHandle(父调子方法)、发布订阅(任意)。

**Q:Context 的缺点?什么时候不该用?**
A:Provider 的 value 变化会导致**所有消费该 Context 的组件重渲染**,粒度粗易引起性能问题。频繁变化的、大范围的状态更适合用 Redux/Zustand(有更细的订阅粒度)。可拆分多个 Context 或配合 memo 缓解。

**Q:为什么 props 是只读的?**
A:保证单向数据流。子组件直接改 props 会破坏数据可预测性;要改必须通知拥有该状态的父组件更新。

**Q:状态提升是什么?**
A:当两个兄弟组件需要共享状态时,把状态移到它们最近的共同父组件,由父组件通过 props 下发和回调更新,实现兄弟间接通信。

**Q:Context 和 Redux 怎么选?**
A:Context 适合低频、局部的共享(主题、语言、登录用户);Redux/Zustand 适合复杂、高频、全局的状态,有中间件、devtools、细粒度订阅等能力。

## 一句话背诵版

**单向数据流:父传子 props、子传父回调、兄弟靠状态提升、跨层级用 Context、全局用 Redux/Zustand、父调子用 ref+useImperativeHandle;Context 的坑是 value 变化会让所有消费者重渲染。**
