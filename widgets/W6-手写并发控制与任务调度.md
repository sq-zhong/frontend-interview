# W6 · 手写并发控制与任务调度

> 业务轮子专项 · 需求 → 设计要点 → 核心实现 → 使用 → 进阶考点

## 需求场景

批量请求(如上传 100 张图、批量查询)不能一次全发出去——会打爆浏览器并发限制、拖垮服务器。需要**并发数控制**:同时最多跑 N 个,完成一个补一个。这是大文件上传(W8)、批处理的基础。

## 设计要点

1. **限制并发数**:维护"正在运行"的计数,达上限则排队;
2. **完成即补位**:一个任务结束,立即从队列取下一个;
3. **收集结果**:按原顺序返回所有结果;
4. 可扩展:失败重试、优先级、暂停/恢复。

## 核心实现

### asyncPool:控制并发数

```js
async function asyncPool(poolLimit, items, iteratorFn) {
  const results = [];       // 保存所有任务的 Promise
  const executing = new Set();  // 正在执行的任务

  for (const [i, item] of items.entries()) {
    // 包装任务
    const p = Promise.resolve().then(() => iteratorFn(item, i));
    results.push(p);

    // 达到并发上限,等待其中一个完成再继续
    if (poolLimit <= items.length) {
      const e = p.then(() => executing.delete(e));
      executing.add(e);
      if (executing.size >= poolLimit) {
        await Promise.race(executing);   // 等最快的一个完成腾出位置
      }
    }
  }
  return Promise.all(results);   // 按原顺序返回所有结果
}
```

### 更直观的队列调度器版本

```js
class TaskQueue {
  constructor(concurrency = 3) {
    this.concurrency = concurrency;
    this.running = 0;
    this.queue = [];
  }

  add(task) {   // task 是返回 Promise 的函数
    return new Promise((resolve, reject) => {
      this.queue.push({ task, resolve, reject });
      this.run();
    });
  }

  run() {
    while (this.running < this.concurrency && this.queue.length) {
      const { task, resolve, reject } = this.queue.shift();
      this.running++;
      task()
        .then(resolve, reject)
        .finally(() => {
          this.running--;
          this.run();   // 完成后补位,取下一个
        });
    }
  }
}
```

## 使用示例

```js
// asyncPool:并发上传,最多同时 3 个
const urls = [/* 100 个图片 */];
const results = await asyncPool(3, urls, (url) => uploadImage(url));

// TaskQueue:动态添加任务
const queue = new TaskQueue(3);
urls.forEach(url => {
  queue.add(() => fetch(url)).then(res => console.log('done', res));
});
```

### 带失败重试的调度

```js
function withRetry(taskFn, retries = 3) {
  return () => taskFn().catch(err => {
    if (retries <= 0) throw err;
    return withRetry(taskFn, retries - 1)();
  });
}
```

## 面试考点

**Q:如何限制并发请求数量?**
A:维护一个"正在执行"的集合和并发上限。每启动一个任务加入集合,达到上限时用 `Promise.race` 等待最快完成的一个腾出位置再继续;或用任务队列,每完成一个就从队列取下一个补位,保持运行数不超过上限。

**Q:Promise.race 在这里的作用?**
A:race 返回最先敲定的 Promise。用它等"正在执行的任务中最快完成的一个",一旦有任务完成就腾出并发名额,继续启动下一个,实现"完成一个补一个"。

**Q:为什么要控制并发?**
A:① 浏览器对同域名 HTTP/1.1 并发有限制(约 6 个),发太多会排队;② 一次性大量请求会瞬间打垮服务器;③ 占用过多内存/带宽。控制并发能平稳处理大批量任务。

**Q:如何保证结果顺序?**
A:虽然任务完成顺序不定,但把每个任务的 Promise 按原顺序存入数组,最后 `Promise.all(results)` 会按数组顺序返回结果,与任务原始顺序一致。

## 一句话总结

**并发控制:维护运行中任务集合 + 上限,达标时用 Promise.race 等最快完成的腾位、完成即补位(asyncPool 或 TaskQueue);按原顺序存 Promise 再 Promise.all 保证结果顺序;可组合 withRetry 失败重试;是大文件上传、批处理的基础,避免打爆浏览器并发限制和服务器。**
