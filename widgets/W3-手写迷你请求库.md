# W3 · 手写迷你请求库(拦截器 / 重试 / 取消 / 缓存)

> 业务轮子专项 · 需求 → 设计要点 → 核心实现 → 使用 → 进阶考点

## 需求场景

项目里对 fetch/axios 二次封装几乎必做:统一加 token、统一错误处理、超时、失败重试、取消、请求去重/缓存。手写一个迷你版能体现对请求全链路的掌控。

## 设计要点

1. **拦截器**:请求拦截(加 token/公共头)、响应拦截(统一处理错误/剥离外层);
2. **超时**:AbortController + setTimeout;
3. **重试**:失败后指数退避重试 N 次(仅幂等请求);
4. **取消**:暴露 cancel;
5. **去重/缓存**:相同请求合并或短时间内返缓存。

## 核心实现

```js
class Request {
  constructor(config = {}) {
    this.baseURL = config.baseURL || '';
    this.timeout = config.timeout || 10000;
    this.interceptors = {
      request: [],
      response: [],
    };
  }

  useRequestInterceptor(fn) { this.interceptors.request.push(fn); }
  useResponseInterceptor(fn) { this.interceptors.response.push(fn); }

  async request(url, options = {}) {
    // 1. 执行请求拦截器(如加 token)
    let config = { url: this.baseURL + url, ...options };
    for (const fn of this.interceptors.request) {
      config = await fn(config);
    }

    // 2. 超时控制
    const controller = new AbortController();
    const timer = setTimeout(() => controller.abort(), this.timeout);

    try {
      let res = await fetch(config.url, {
        ...config,
        signal: config.signal || controller.signal,
      });
      clearTimeout(timer);

      if (!res.ok) throw new Error(`HTTP ${res.status}`);
      let data = await res.json();

      // 3. 执行响应拦截器(如剥离外层、统一错误码)
      for (const fn of this.interceptors.response) {
        data = await fn(data);
      }
      return data;
    } catch (err) {
      clearTimeout(timer);
      throw err;
    }
  }

  get(url, options) { return this.request(url, { ...options, method: 'GET' }); }
  post(url, body, options) {
    return this.request(url, {
      ...options, method: 'POST',
      headers: { 'Content-Type': 'application/json', ...options?.headers },
      body: JSON.stringify(body),
    });
  }
}
```

### 失败重试(指数退避)

```js
async function withRetry(fn, retries = 3, delay = 500) {
  try {
    return await fn();
  } catch (err) {
    if (retries <= 0) throw err;
    await new Promise(r => setTimeout(r, delay));
    return withRetry(fn, retries - 1, delay * 2);   // 退避:500→1000→2000
  }
}
```

### 请求去重(合并并发的相同请求)

```js
const pending = new Map();
function dedupe(key, fn) {
  if (pending.has(key)) return pending.get(key);   // 复用在途请求
  const promise = fn().finally(() => pending.delete(key));
  pending.set(key, promise);
  return promise;
}
```

## 使用示例

```js
const http = new Request({ baseURL: '/api', timeout: 8000 });

// 请求拦截:加 token
http.useRequestInterceptor((config) => {
  config.headers = { ...config.headers, Authorization: `Bearer ${getToken()}` };
  return config;
});

// 响应拦截:统一处理业务码
http.useResponseInterceptor((data) => {
  if (data.code !== 0) throw new Error(data.message);
  return data.data;   // 剥离外层,直接返回业务数据
});

const users = await withRetry(() => http.get('/users'));
```

## 面试考点

**Q:拦截器的原理?**
A:维护请求/响应两个拦截器数组,发请求前依次执行请求拦截器(改 config,如加 token),拿到响应后依次执行响应拦截器(处理数据,如统一错误码、剥离外层)。本质是责任链/中间件模式。

**Q:如何实现超时和取消?**
A:用 AbortController,把 signal 传给 fetch;超时用 setTimeout 到点调 `controller.abort()` 中断请求;主动取消则暴露 controller 让业务调 abort。

**Q:什么请求可以自动重试?**
A:只有幂等请求(GET、PUT、DELETE)可安全重试,POST 等非幂等请求重试可能导致重复提交/重复扣款。重试用指数退避避免雪崩。

**Q:请求去重怎么做?**
A:用请求特征(method+url+params)做 key,存一个 Map 记录在途 Promise;相同 key 的并发请求直接复用同一个 Promise,完成后从 Map 删除。避免短时间重复请求。

## 一句话总结

**迷你请求库:请求/响应拦截器(责任链,加 token/统一错误)、AbortController 实现超时与取消、指数退避重试(仅幂等请求)、Map 缓存在途 Promise 做去重;封装 get/post 简化调用;本质是对请求全链路的中间件式管控。**
