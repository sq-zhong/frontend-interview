# W2 · 手写虚拟列表组件

> 业务轮子专项 · 需求 → 设计要点 → 核心实现 → 使用 → 进阶考点

## 需求场景

渲染上万条数据的列表,全量渲染会产生上万 DOM,首屏慢、滚动卡、内存高。虚拟列表**只渲染可视区域内的少量 DOM**,是长列表性能优化的标准方案(见 B7)。

## 设计要点

1. **占位撑高**:用一个容器撑起 `总条数 × 行高` 的高度,保证滚动条正确;
2. **计算可见区间**:根据 `scrollTop` 算出应渲染的起止索引;
3. **偏移定位**:用 `transform: translateY` 把可见内容推到正确位置;
4. **缓冲区(overscan)**:上下多渲染几条,减少快速滚动时的白屏;
5. 定高简单,**不定高**需测量缓存。

## 核心实现(定高版)

```jsx
import { useState, useRef } from 'react';

function VirtualList({ data, itemHeight = 50, height = 500, overscan = 3 }) {
  const [scrollTop, setScrollTop] = useState(0);
  const containerRef = useRef(null);

  const total = data.length;
  // 可见起止索引(带缓冲)
  const startIndex = Math.max(0, Math.floor(scrollTop / itemHeight) - overscan);
  const visibleCount = Math.ceil(height / itemHeight) + overscan * 2;
  const endIndex = Math.min(total, startIndex + visibleCount);

  const visibleData = data.slice(startIndex, endIndex);
  const offsetY = startIndex * itemHeight;   // 可见内容的偏移

  return (
    <div
      ref={containerRef}
      style={{ height, overflow: 'auto', position: 'relative' }}
      onScroll={e => setScrollTop(e.currentTarget.scrollTop)}
    >
      {/* 占位:撑起总高度,滚动条才正确 */}
      <div style={{ height: total * itemHeight }} />
      {/* 可见内容:绝对定位 + translateY 推到正确位置 */}
      <div style={{
        position: 'absolute', top: 0, left: 0, right: 0,
        transform: `translateY(${offsetY}px)`,
      }}>
        {visibleData.map((item, i) => (
          <div key={startIndex + i} style={{ height: itemHeight }}>
            {item.text}
          </div>
        ))}
      </div>
    </div>
  );
}
```

## 使用示例

```jsx
const data = Array.from({ length: 100000 }, (_, i) => ({ text: `Item ${i}` }));

<VirtualList data={data} itemHeight={50} height={600} />
// 10 万条数据,DOM 里始终只有约 15 个节点
```

## 进阶:不定高列表

```jsx
// 思路:预估高度 → 渲染后测量真实高度并缓存 → 修正后续偏移
// 1. 维护 measuredHeights = Map<index, height>
// 2. 用累加的位置数组(positions)记录每条的 top
// 3. 二分查找 scrollTop 落在哪一条(startIndex)
// 4. 渲染后用 ResizeObserver / ref 测量真实高度,更新缓存和 positions

function getStartIndex(positions, scrollTop) {
  // positions 单调递增,二分找第一个 >= scrollTop 的
  let lo = 0, hi = positions.length - 1;
  while (lo < hi) {
    const mid = (lo + hi) >> 1;
    if (positions[mid] < scrollTop) lo = mid + 1;
    else hi = mid;
  }
  return lo;
}
```

## 面试考点

**Q:虚拟列表的核心原理?**
A:只渲染可视区域内的 DOM。用一个占位元素撑起 `总条数×行高` 的总高度让滚动条正常;监听 scrollTop 算出当前应渲染的起止索引;用 translateY 把这批可见内容偏移到正确位置。滚动时动态更新索引和偏移,DOM 数量恒定。

**Q:为什么用 transform 而不是改 top/margin 定位?**
A:transform 走 GPU 合成层,不触发重排(见 H3),滚动时性能更好、更流畅。

**Q:不定高怎么处理?**
A:预估初始高度先渲染,渲染后用 ref/ResizeObserver 测量真实高度并缓存,维护每条的累计位置数组;用二分查找根据 scrollTop 定位起始索引。复杂度高,可用 react-window / react-virtualized 的 VariableSizeList。

**Q:overscan(缓冲区)的作用?**
A:在可视区上下多渲染几条,快速滚动时新内容已提前就位,避免边缘白屏闪烁,用少量额外 DOM 换视觉流畅。

## 一句话总结

**虚拟列表只渲染可视区 DOM:占位元素撑总高度(总数×行高)保证滚动条、按 scrollTop 算起止索引、用 translateY(走 GPU)偏移可见内容、overscan 缓冲防白屏;定高直接算,不定高用预估+测量缓存+二分定位;DOM 数量恒定,10 万条也只渲染十几个。**
