# W4 · 手写迷你状态管理

> 业务轮子专项 · 需求 → 设计要点 → 核心实现 → 使用 → 进阶考点

## 需求场景

跨组件共享状态。手写一个类 Zustand 的迷你状态管理(发布订阅 + Hook),能讲清"状态如何驱动组件更新"的本质,比背 Redux API 更有价值。

## 设计要点

1. **单一 store + 发布订阅**:state 变化时通知订阅者;
2. **set/get**:更新和读取状态;
3. **Hook 订阅**:组件用 hook 订阅,状态变化时重渲染;
4. **选择器(selector)**:只订阅用到的字段,精准更新(性能关键);
5. 用 `useSyncExternalStore`(React 18 官方推荐,解决并发安全)。

## 核心实现

```js
import { useSyncExternalStore } from 'react';

function createStore(createState) {
  let state;
  const listeners = new Set();

  // 更新状态:合并 + 通知所有订阅者
  const setState = (partial) => {
    const nextState = typeof partial === 'function' ? partial(state) : partial;
    state = { ...state, ...nextState };
    listeners.forEach(listener => listener());   // 发布
  };

  const getState = () => state;

  // 订阅:返回取消订阅函数
  const subscribe = (listener) => {
    listeners.add(listener);
    return () => listeners.delete(listener);
  };

  // 初始化 state(把 set/get 传给用户的创建函数)
  state = createState(setState, getState);

  // 返回一个 hook:用 selector 精准订阅
  const useStore = (selector = (s) => s) => {
    return useSyncExternalStore(
      subscribe,
      () => selector(state),        // 只取需要的部分
      () => selector(state)         // SSR 快照
    );
  };

  useStore.setState = setState;
  useStore.getState = getState;
  useStore.subscribe = subscribe;
  return useStore;
}
```

## 使用示例

```jsx
// 创建 store(和 Zustand 用法几乎一致)
const useCounterStore = createStore((set, get) => ({
  count: 0,
  inc: () => set(s => ({ count: s.count + 1 })),
  incBy: (n) => set(s => ({ count: s.count + n })),
  reset: () => set({ count: 0 }),
}));

function Counter() {
  // selector 只订阅 count,其他字段变化不会导致这里重渲染
  const count = useCounterStore(s => s.count);
  const inc = useCounterStore(s => s.inc);
  return <button onClick={inc}>{count}</button>;
}

// 组件外也能用
useCounterStore.getState().inc();
useCounterStore.subscribe(() => console.log('state changed'));
```

## 面试考点

**Q:状态管理的本质是什么?**
A:发布订阅模式。一个集中的 store 保存状态,组件订阅 store;当通过 setState 修改状态时,遍历通知所有订阅者(组件)触发重渲染。selector 让组件只订阅关心的部分,实现精准更新。

**Q:为什么用 useSyncExternalStore?**
A:它是 React 18 为"订阅外部 store"提供的官方 Hook,解决了并发渲染下的**撕裂问题**(tearing,不同组件读到不一致的状态)和 SSR 一致性。手动用 useState+useEffect 订阅在并发模式下可能不安全。

**Q:selector 如何实现精准更新?**
A:组件传入 selector 只取需要的字段。useSyncExternalStore 会比较 selector 返回值,只有该部分变化时才触发重渲染,避免"store 任意字段变化都重渲染所有组件"。

**Q:这和 Redux 有什么区别?**
A:Redux 强调单向数据流、reducer 纯函数、action 描述变更,样板多;这种(Zustand 风格)直接 set 改状态、无 action/reducer、hook 式订阅,更轻量。核心机制都是发布订阅。

## 一句话总结

**迷你状态管理 = 发布订阅:store 存 state + listeners 集合,setState 合并状态后通知所有订阅者;用 useSyncExternalStore 订阅(解决并发撕裂和 SSR)、传 selector 精准订阅需要的字段避免无效重渲染;本质与 Redux/Zustand 一致,只是 API 繁简不同。**
