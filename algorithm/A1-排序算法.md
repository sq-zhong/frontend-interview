# A1 · 排序算法

> 热度:🔥🔥

## 一句话考点

面试高频:**快排**(平均 O(n log n)、分治)、**归并**(稳定 O(n log n)、分治)、**冒泡/插入/选择**(O(n²));要能手写快排并说清时间复杂度和稳定性。

## 原理精讲

### 复杂度与稳定性对比

| 算法 | 平均 | 最坏 | 空间 | 稳定 |
|------|------|------|------|------|
| 冒泡 | O(n²) | O(n²) | O(1) | ✅ |
| 选择 | O(n²) | O(n²) | O(1) | ❌ |
| 插入 | O(n²) | O(n²) | O(1) | ✅ |
| **快排** | O(n log n) | O(n²) | O(log n) | ❌ |
| **归并** | O(n log n) | O(n log n) | O(n) | ✅ |
| 堆排序 | O(n log n) | O(n log n) | O(1) | ❌ |

**稳定**:相等元素的相对顺序不变。

### 快速排序思路

分治:选一个基准(pivot),把数组分成"小于基准"和"大于基准"两部分,递归排序两部分。

### 归并排序思路

分治:把数组不断二分到单元素,再两两**有序合并**回去。

## 代码示例

### 快速排序

```js
// 简洁版(非原地,易理解)
function quickSort(arr) {
  if (arr.length <= 1) return arr;
  const pivot = arr[Math.floor(arr.length / 2)];
  const left = [], right = [], equal = [];
  for (const n of arr) {
    if (n < pivot) left.push(n);
    else if (n > pivot) right.push(n);
    else equal.push(n);
  }
  return [...quickSort(left), ...equal, ...quickSort(right)];
}

// 原地版(省空间,面试更佳)
function quickSortInPlace(arr, lo = 0, hi = arr.length - 1) {
  if (lo >= hi) return arr;
  const pivot = arr[hi];
  let i = lo;
  for (let j = lo; j < hi; j++) {
    if (arr[j] < pivot) {
      [arr[i], arr[j]] = [arr[j], arr[i]];
      i++;
    }
  }
  [arr[i], arr[hi]] = [arr[hi], arr[i]];   // 基准归位
  quickSortInPlace(arr, lo, i - 1);
  quickSortInPlace(arr, i + 1, hi);
  return arr;
}
```

### 归并排序

```js
function mergeSort(arr) {
  if (arr.length <= 1) return arr;
  const mid = arr.length >> 1;
  const left = mergeSort(arr.slice(0, mid));
  const right = mergeSort(arr.slice(mid));
  return merge(left, right);
}
function merge(left, right) {
  const res = [];
  let i = 0, j = 0;
  while (i < left.length && j < right.length) {
    // 用 <= 保证稳定性
    if (left[i] <= right[j]) res.push(left[i++]);
    else res.push(right[j++]);
  }
  return res.concat(left.slice(i)).concat(right.slice(j));
}
```

## 高频追问

**Q:快排的时间复杂度?最坏情况?**
A:平均 O(n log n)。最坏 O(n²),发生在每次基准都选到最大/最小(如已排序数组选端点为基准)。可用随机基准或三数取中优化避免最坏。

**Q:快排和归并的区别?**
A:快排原地排序(空间 O(log n)递归栈)、不稳定、平均更快常数小;归并需 O(n) 额外空间、稳定、最坏也是 O(n log n)。大数据外部排序常用归并,内存排序常用快排。

**Q:哪些排序是稳定的?**
A:冒泡、插入、归并稳定;选择、快排、堆排序不稳定。稳定指相等元素排序后相对位置不变,涉及多关键字排序时重要。

**Q:JS 的 Array.sort 用什么算法?**
A:V8 中,元素少时用插入排序,多时用 TimSort(归并+插入的混合,稳定)。注意默认按字符串比较,数字排序要传比较函数 `sort((a,b)=>a-b)`。

**Q:如何优化快排避免最坏情况?**
A:① 随机选基准;② 三数取中(取首、中、尾的中位数);③ 小数组切换插入排序;④ 三路快排处理大量重复元素。

## 一句话背诵版

**快排(分治、平均 O(n log n)、最坏 O(n²)、原地不稳定,随机基准优化)、归并(分治、稳定 O(n log n)、需 O(n) 空间);冒泡/插入稳定、选择/快排/堆不稳定;JS sort 是稳定 TimSort、数字排序要传比较函数。**
