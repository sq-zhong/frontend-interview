# A8 · 回溯与经典高频题

> 热度:🔥🔥

## 一句话考点

**回溯**是带撤销的 DFS,用于枚举所有可能(排列、组合、子集、路径);模板:**选择 → 递归 → 撤销选择**;同时汇总前端最常考的算法题。

## 原理精讲

### 1. 回溯框架

```
result = []
function backtrack(路径, 选择列表):
    if 满足结束条件:
        result.push(路径的拷贝)
        return
    for 选择 in 选择列表:
        做选择(路径.push)
        backtrack(路径, 新选择列表)
        撤销选择(路径.pop)      ← 关键:回溯
```

回溯 = DFS + 状态撤销,本质是在决策树上遍历所有路径。

### 2. 三类问题去重

- **组合/子集**:用 `start` 索引避免重复选(不回头);
- **排列**:用 `used[]` 标记已用元素;
- **有重复元素**:先排序,同层跳过相同值 `if (i > start && nums[i] === nums[i-1]) continue`。

## 代码示例

### 全排列

```js
function permute(nums) {
  const res = [], path = [], used = new Array(nums.length).fill(false);
  function backtrack() {
    if (path.length === nums.length) {
      res.push([...path]);              // 拷贝当前路径
      return;
    }
    for (let i = 0; i < nums.length; i++) {
      if (used[i]) continue;            // 跳过已用
      path.push(nums[i]); used[i] = true;    // 选择
      backtrack();                            // 递归
      path.pop(); used[i] = false;            // 撤销
    }
  }
  backtrack();
  return res;
}
```

### 子集

```js
function subsets(nums) {
  const res = [], path = [];
  function backtrack(start) {
    res.push([...path]);                // 每个节点都是一个子集
    for (let i = start; i < nums.length; i++) {
      path.push(nums[i]);
      backtrack(i + 1);                 // start=i+1 保证不回头,避免重复
      path.pop();
    }
  }
  backtrack(0);
  return res;
}
```

### 组合总和

```js
function combinationSum(candidates, target) {
  const res = [], path = [];
  function backtrack(start, remain) {
    if (remain === 0) { res.push([...path]); return; }
    if (remain < 0) return;
    for (let i = start; i < candidates.length; i++) {
      path.push(candidates[i]);
      backtrack(i, remain - candidates[i]);   // 可重复选,传 i 不是 i+1
      path.pop();
    }
  }
  backtrack(0, target);
  return res;
}
```

### 有效的括号(栈)

```js
function isValid(s) {
  const stack = [], map = { ')': '(', ']': '[', '}': '{' };
  for (const c of s) {
    if (c === '(' || c === '[' || c === '{') stack.push(c);
    else if (stack.pop() !== map[c]) return false;   // 不匹配
  }
  return stack.length === 0;
}
```

## 高频追问

**Q:回溯的模板?**
A:做选择 → 递归进入下一层 → 撤销选择。在决策树上 DFS,到达结束条件时记录路径拷贝。核心是递归后要撤销(pop)以尝试其他分支。

**Q:回溯和 DFS 的关系?**
A:回溯是 DFS 的一种,特点是在递归返回后**撤销状态**(恢复现场),以便探索其他分支。普通 DFS(如遍历)不一定需要撤销。

**Q:组合和排列的去重区别?**
A:组合用 start 索引控制只向后选(不回头),天然去重顺序;排列用 used 数组标记已选元素(可回头但不重复用同一个)。有重复元素时先排序再同层跳过相同值。

**Q:为什么记录结果要拷贝路径 `[...path]`?**
A:path 是同一个数组引用,回溯过程中会被增删。直接 push(path) 会导致所有结果指向同一个最终为空的数组,必须存快照拷贝。

**Q:回溯的时间复杂度?**
A:通常指数级或阶乘级(枚举所有可能):子集 O(2ⁿ)、排列 O(n!)。因为要遍历整棵决策树。可通过剪枝(提前 return 不合法分支)减少实际计算。

## 一句话背诵版

**回溯 = DFS + 撤销:做选择→递归→撤销,决策树上枚举所有可能(排列用 used、组合/子集用 start、重复元素排序后同层跳过);记录结果要拷贝路径 `[...path]`;高频题速记:两数之和(Map)、括号匹配(栈)、反转链表、层序(BFS)、爬楼梯(DP)、全排列(回溯)。**
