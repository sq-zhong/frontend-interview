# A2 · 二分查找

> 热度:🔥🔥

## 一句话考点

二分查找在**有序数组**中每次折半缩小范围,时间 O(log n);难点在**边界处理**(区间开闭、`mid` 计算、`left/right` 更新),以及查找边界(第一个/最后一个)的变体。

## 原理精讲

### 1. 基本思想

有序数组中,比较中间元素与目标:
- 相等 → 找到;
- 目标更小 → 去左半区;
- 目标更大 → 去右半区。

每次排除一半,O(log n)。

### 2. 边界处理关键

- 循环条件 `left <= right`(闭区间)还是 `left < right`(半开);
- `mid = left + ((right - left) >> 1)`(防溢出);
- 更新时 `left = mid + 1` / `right = mid - 1`(闭区间)。

保持**区间定义一致**是不出错的关键。

### 3. 常见变体

- 查找目标(标准);
- 查找**第一个**等于 target 的位置(左边界);
- 查找**最后一个**等于 target 的位置(右边界);
- 查找第一个 ≥ target(lower_bound);
- 旋转数组中查找。

## 代码示例

### 标准二分(闭区间)

```js
function binarySearch(nums, target) {
  let left = 0, right = nums.length - 1;   // 闭区间 [left, right]
  while (left <= right) {
    const mid = left + ((right - left) >> 1);
    if (nums[mid] === target) return mid;
    else if (nums[mid] < target) left = mid + 1;
    else right = mid - 1;
  }
  return -1;
}
```

### 查找左边界(第一个 ≥ target)

```js
function lowerBound(nums, target) {
  let left = 0, right = nums.length;       // 半开 [left, right)
  while (left < right) {
    const mid = left + ((right - left) >> 1);
    if (nums[mid] < target) left = mid + 1;
    else right = mid;                       // 保留 mid,继续向左找
  }
  return left;   // 第一个 >= target 的下标(可能等于 length)
}
```

### 搜索旋转排序数组(高频)

```js
function search(nums, target) {
  let left = 0, right = nums.length - 1;
  while (left <= right) {
    const mid = (left + right) >> 1;
    if (nums[mid] === target) return mid;
    if (nums[left] <= nums[mid]) {          // 左半有序
      if (nums[left] <= target && target < nums[mid]) right = mid - 1;
      else left = mid + 1;
    } else {                                 // 右半有序
      if (nums[mid] < target && target <= nums[right]) left = mid + 1;
      else right = mid - 1;
    }
  }
  return -1;
}
```

## 高频追问

**Q:二分查找的前提和复杂度?**
A:前提是数组**有序**(或有单调性)。时间 O(log n),空间 O(1)(迭代)。每次比较排除一半元素。

**Q:`mid = (left + right) / 2` 有什么问题?**
A:当 left、right 很大时 `left + right` 可能整数溢出(其他语言)。写成 `left + ((right - left) >> 1)` 更安全。JS 数字范围大,溢出风险低,但仍推荐这种写法。

**Q:如何查找第一个/最后一个等于 target 的位置?**
A:找到 target 后不立即返回,而是继续向左(找第一个)或向右(找最后一个)收缩。或用 lower_bound(第一个 ≥ target)和 upper_bound(第一个 > target)组合。

**Q:二分的边界怎么不出错?**
A:先明确区间定义(闭区间 [l,r] 还是半开 [l,r)),然后循环条件、mid 归属、指针更新都严格按定义来。闭区间用 `l<=r` 和 `mid±1`;半开用 `l<r` 和 `right=mid`。

**Q:旋转数组二分的思路?**
A:每次判断 mid 落在哪一半(哪半是有序的),再判断 target 是否在有序的那半区间内,决定往哪边收缩。总有一半是有序的。

## 一句话背诵版

**二分要求有序,O(log n),每次折半;`mid = left + ((right-left)>>1)` 防溢出;边界靠统一区间定义(闭区间 l<=r 配 mid±1);变体:左/右边界(找到不返回继续收缩)、lower_bound、旋转数组(判断哪半有序再收缩)。**
