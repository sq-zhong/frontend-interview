# A4 · 链表

> 热度:🔥🔥🔥

## 一句话考点

链表由节点(值 + 指针)串联,增删 O(1)、随机访问 O(n);高频题:**反转链表、判环、找中点、合并有序链表、删除倒数第 N 个**,核心技巧是**虚拟头节点**和**快慢/双指针**。

## 原理精讲

### 1. 链表 vs 数组

| | 数组 | 链表 |
|--|------|------|
| 随机访问 | O(1) | O(n) |
| 头部增删 | O(n) | O(1) |
| 内存 | 连续 | 分散(指针开销) |

### 2. 常用技巧

- **虚拟头节点(dummy)**:`dummy.next = head`,统一处理头节点的增删,避免特判;
- **双指针/快慢指针**:找中点、判环、找倒数第 N 个;
- **画图**:反转类题目务必画出指针指向。

## 代码示例

### 反转链表(必考)

```js
// 迭代:三指针
function reverseList(head) {
  let prev = null, cur = head;
  while (cur) {
    const next = cur.next;   // 暂存下一个
    cur.next = prev;         // 反转指针
    prev = cur;              // prev 前进
    cur = next;              // cur 前进
  }
  return prev;               // 新头
}

// 递归
function reverseRecursive(head) {
  if (!head || !head.next) return head;
  const newHead = reverseRecursive(head.next);
  head.next.next = head;     // 反转
  head.next = null;
  return newHead;
}
```

### 判断链表有环 + 找入口

```js
function detectCycle(head) {
  let slow = head, fast = head;
  while (fast && fast.next) {
    slow = slow.next;
    fast = fast.next.next;
    if (slow === fast) {              // 相遇 → 有环
      let p = head;
      while (p !== slow) { p = p.next; slow = slow.next; }
      return p;                       // 环入口
    }
  }
  return null;                        // 无环
}
```

### 合并两个有序链表

```js
function mergeTwoLists(l1, l2) {
  const dummy = new ListNode(0);      // 虚拟头简化逻辑
  let cur = dummy;
  while (l1 && l2) {
    if (l1.val <= l2.val) { cur.next = l1; l1 = l1.next; }
    else { cur.next = l2; l2 = l2.next; }
    cur = cur.next;
  }
  cur.next = l1 || l2;                // 接上剩余
  return dummy.next;
}
```

### 删除倒数第 N 个节点

```js
function removeNthFromEnd(head, n) {
  const dummy = new ListNode(0, head);
  let fast = dummy, slow = dummy;
  for (let i = 0; i < n; i++) fast = fast.next;   // fast 先走 n 步
  while (fast.next) { fast = fast.next; slow = slow.next; }
  slow.next = slow.next.next;         // 删除
  return dummy.next;
}
```

## 高频追问

**Q:如何反转链表?**
A:迭代法用三个指针 prev/cur/next:暂存 next,把 cur.next 指向 prev,然后 prev 和 cur 各前进,直到 cur 为 null,prev 即新头。O(n) 时间 O(1) 空间。也可递归。

**Q:虚拟头节点的作用?**
A:在链表头前加一个 dummy 节点,让头节点的插入/删除和其他节点逻辑统一,避免对"操作头节点"单独特判,简化代码。返回时用 dummy.next。

**Q:如何判断链表是否有环?找环入口?**
A:快慢指针,快走两步慢走一步。相遇则有环。找入口:相遇后一个指针回到头,与相遇点指针同速前进,再次相遇处即环入口(基于数学推导)。

**Q:链表和数组如何选择?**
A:频繁随机访问用数组(O(1));频繁在头部/中间插入删除用链表(O(1),无需搬移);链表有指针内存开销且缓存不友好。

**Q:如何找链表中点?**
A:快慢指针,快指针每次两步、慢指针一步,快指针到尾时慢指针在中点。偶数长度取决于 fast 初始位置和循环条件(返回中间偏左或偏右)。

## 一句话背诵版

**链表增删 O(1)、随机访问 O(n);反转用三指针(暂存 next→cur.next 指 prev→前进);判环用快慢指针相遇、入口靠头指针与相遇点同速再遇;虚拟头节点统一头部增删;找中点/倒数第 N 个用快慢指针。**
