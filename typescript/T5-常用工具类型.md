# T5 · 常用工具类型

> 热度:🔥🔥🔥

## 一句话考点

TS 内置工具类型(Utility Types)基于映射/条件类型实现,高频:`Partial`、`Required`、`Readonly`、`Pick`、`Omit`、`Record`、`Exclude`、`Extract`、`ReturnType`、`Parameters`。

## 原理精讲

### 1. 对象属性修饰

| 工具类型 | 作用 |
|----------|------|
| `Partial<T>` | 所有属性变可选 |
| `Required<T>` | 所有属性变必选 |
| `Readonly<T>` | 所有属性变只读 |

### 2. 属性筛选

| 工具类型 | 作用 |
|----------|------|
| `Pick<T, K>` | 从 T 挑选 K 指定的属性 |
| `Omit<T, K>` | 从 T 剔除 K 指定的属性 |
| `Record<K, V>` | 构造键为 K、值为 V 的对象类型 |

### 3. 联合类型操作

| 工具类型 | 作用 |
|----------|------|
| `Exclude<T, U>` | 从联合 T 中排除 U |
| `Extract<T, U>` | 从联合 T 中提取 U |
| `NonNullable<T>` | 排除 null 和 undefined |

### 4. 函数相关

| 工具类型 | 作用 |
|----------|------|
| `ReturnType<F>` | 获取函数返回值类型 |
| `Parameters<F>` | 获取函数参数类型(元组) |
| `Awaited<T>` | 获取 Promise 解析后的类型 |

## 代码示例

### 对象属性修饰

```ts
interface User { id: number; name: string; age: number; }

type PartialUser = Partial<User>;    // 所有属性可选(常用于更新接口)
// { id?: number; name?: string; age?: number }

type ReadonlyUser = Readonly<User>;  // 所有属性只读

// 实际:更新函数只传部分字段
function updateUser(id: number, data: Partial<User>) {}
updateUser(1, { name: 'Tom' });      // ✅ 只传部分
```

### Pick / Omit / Record

```ts
type UserPreview = Pick<User, 'id' | 'name'>;   // { id; name }
type UserWithoutId = Omit<User, 'id'>;           // { name; age }

// Record 构造映射对象
type Roles = Record<'admin' | 'guest', string[]>;
// { admin: string[]; guest: string[] }

const pageMap: Record<string, number> = { home: 1, about: 2 };
```

### 联合与函数工具

```ts
type T = Exclude<'a' | 'b' | 'c', 'a'>;   // 'b' | 'c'
type E = Extract<'a' | 'b', 'a' | 'x'>;   // 'a'
type N = NonNullable<string | null>;       // string

function fn(a: number, b: string) { return { a, b }; }
type R = ReturnType<typeof fn>;            // { a: number; b: string }
type P = Parameters<typeof fn>;            // [number, string]
type A = Awaited<Promise<string>>;         // string
```

### 部分工具类型的实现(理解原理)

```ts
// Partial 的实现:映射类型 + 可选修饰符
type MyPartial<T> = { [K in keyof T]?: T[K] };

// Pick 的实现
type MyPick<T, K extends keyof T> = { [P in K]: T[P] };

// Exclude 的实现:条件类型 + 分布式
type MyExclude<T, U> = T extends U ? never : T;

// Record 的实现
type MyRecord<K extends keyof any, V> = { [P in K]: V };
```

## 高频追问

**Q:Partial 和 Required 怎么实现?**
A:用映射类型遍历键。Partial 加可选修饰 `?`:`{ [K in keyof T]?: T[K] }`;Required 用 `-?` 去掉可选:`{ [K in keyof T]-?: T[K] }`。

**Q:Pick 和 Omit 区别?**
A:Pick<T, K> 从 T 中**保留** K 指定的属性;Omit<T, K> 从 T 中**剔除** K 指定的属性。Omit 内部用 Pick + Exclude 实现:`Pick<T, Exclude<keyof T, K>>`。

**Q:Record 的用途?**
A:构造键值类型统一的对象类型。如 `Record<string, number>` 表示任意字符串键、值为 number 的对象;`Record<'a'|'b', T>` 构造固定键的映射,常用于字典、枚举映射。

**Q:Exclude 和 Omit 区别?**
A:Exclude 作用于**联合类型**(排除某些成员);Omit 作用于**对象类型**(排除某些属性)。Omit 内部借助 Exclude 来筛选键。

**Q:ReturnType 怎么拿到函数返回类型?**
A:基于条件类型 + infer:`type ReturnType<T> = T extends (...args: any) => infer R ? R : never`,用 infer 推断出返回值类型 R(见 T7)。

## 一句话背诵版

**高频工具类型:Partial/Required/Readonly(改属性修饰)、Pick/Omit/Record(筛选/构造对象)、Exclude/Extract/NonNullable(联合操作)、ReturnType/Parameters/Awaited(函数与 Promise);底层靠映射类型(`[K in keyof T]`)和条件类型(`extends ? :` + infer)实现。**
