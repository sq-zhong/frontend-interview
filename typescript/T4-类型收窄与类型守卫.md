# T4 · 类型收窄与类型守卫

> 热度:🔥🔥

## 一句话考点

类型收窄(narrowing)是把宽泛类型(联合/unknown)缩小到具体类型;靠**类型守卫**实现:`typeof`、`instanceof`、`in`、字面量判断、自定义守卫 `is`。

## 原理精讲

### 1. 为什么要收窄

联合类型或 unknown 无法直接访问特定成员,需要先判断具体是哪种类型,TS 才允许安全访问对应属性/方法。

### 2. 内置类型守卫

| 守卫 | 用途 |
|------|------|
| `typeof x === 'string'` | 判断原始类型 |
| `x instanceof Class` | 判断类实例 |
| `'key' in obj` | 判断对象是否有某属性 |
| `x === '字面量'` | 字面量联合收窄 |
| `Array.isArray(x)` | 判断数组 |

### 3. 自定义类型守卫(`is`)

用返回 `参数 is 类型` 的函数,把复杂判断封装成守卫:

```ts
function isString(val: unknown): val is string {
  return typeof val === 'string';
}
```

### 4. 可辨识联合(Discriminated Union)

给联合类型每个成员加一个**公共字面量字段**(如 `type`/`kind`),用它来收窄,是处理联合类型的最佳实践。

## 代码示例

### typeof / in 收窄

```ts
function format(val: string | number) {
  if (typeof val === 'string') {
    return val.toUpperCase();   // 这里 val 被收窄为 string
  }
  return val.toFixed(2);        // 否则是 number
}

// in 收窄对象联合
type Cat = { meow(): void };
type Dog = { bark(): void };
function speak(animal: Cat | Dog) {
  if ('meow' in animal) animal.meow();   // 收窄为 Cat
  else animal.bark();                     // 收窄为 Dog
}
```

### 自定义守卫

```ts
interface User { name: string; }
function isUser(val: unknown): val is User {
  return typeof val === 'object' && val !== null && 'name' in val;
}

function greet(val: unknown) {
  if (isUser(val)) {
    console.log(val.name);   // ✅ val 被收窄为 User
  }
}
```

### 可辨识联合(推荐)

```ts
type Shape =
  | { kind: 'circle'; radius: number }
  | { kind: 'rect'; width: number; height: number };

function area(shape: Shape): number {
  switch (shape.kind) {          // 用公共字段 kind 收窄
    case 'circle':
      return Math.PI * shape.radius ** 2;   // 这里能访问 radius
    case 'rect':
      return shape.width * shape.height;    // 这里能访问 width/height
    default:
      const _: never = shape;    // 穷尽检查
      return _;
  }
}
```

## 高频追问

**Q:什么是类型收窄?**
A:把宽泛类型(联合类型、unknown)通过条件判断缩小到更具体的类型,使 TS 允许安全访问该具体类型的成员。靠类型守卫实现。

**Q:有哪些类型守卫?**
A:内置的 typeof(原始类型)、instanceof(类实例)、in(属性存在)、字面量相等判断、Array.isArray;以及自定义守卫函数(返回 `x is T`)。

**Q:自定义类型守卫怎么写?**
A:函数返回类型写成 `参数 is 目标类型`,函数体返回布尔值判断。调用后 TS 会在 true 分支把参数收窄为目标类型。适合封装复杂判断逻辑。

**Q:什么是可辨识联合?为什么推荐?**
A:联合类型的每个成员共享一个字面量字段(如 kind/type)作为"标签"。用 switch 判断该字段即可收窄到对应成员并安全访问其独有属性。它清晰、类型安全,配合 never 可做穷尽检查。

**Q:in 操作符收窄的注意点?**
A:`'prop' in obj` 判断对象是否包含某属性来收窄联合类型。注意属性名要是各成员的差异字段,若属性可选或多成员都有则无法有效区分。

## 一句话背诵版

**类型收窄把联合/unknown 缩小到具体类型;守卫:typeof(原始)、instanceof(实例)、in(属性)、字面量判断、Array.isArray、自定义 `x is T`;联合类型最佳实践是可辨识联合(公共 kind 字段 + switch 收窄 + never 穷尽检查)。**
