# T3 · 泛型

> 热度:🔥🔥🔥

## 一句话考点

泛型是**类型的参数化**——把类型当参数传入,让函数/接口/类能处理多种类型且保持类型安全;用 `extends` 约束泛型、`=` 设默认值。

## 原理精讲

### 1. 为什么需要泛型

不用泛型时,复用函数要么用 any(失去类型),要么为每种类型重写。泛型让类型"延迟到调用时确定",既复用又保留类型信息。

```ts
// ❌ any 失去类型
function identity(arg: any): any { return arg; }

// ✅ 泛型:输入输出类型关联,保留类型
function identity<T>(arg: T): T { return arg; }
identity<string>('a');   // 显式指定
identity(1);             // 自动推断 T = number
```

### 2. 泛型约束(extends)

限制泛型必须满足某种结构:

```ts
// T 必须有 length 属性
function logLength<T extends { length: number }>(arg: T): T {
  console.log(arg.length);
  return arg;
}
logLength('abc');      // ✅ string 有 length
logLength([1, 2]);     // ✅ 数组有 length
// logLength(123);     // ❌ number 没有 length
```

### 3. 泛型默认值

```ts
interface Box<T = string> {
  value: T;
}
const b: Box = { value: 'a' };   // 不传时默认 string
```

### 4. keyof 约束(常见组合)

```ts
// K 被约束为 T 的键名,保证 key 合法且返回值类型精确
function getProp<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}
const user = { name: 'Tom', age: 18 };
getProp(user, 'name');   // 返回类型 string
getProp(user, 'age');    // 返回类型 number
// getProp(user, 'xxx'); // ❌ 'xxx' 不是 user 的键
```

## 代码示例

### 泛型接口与类

```ts
// 泛型接口
interface ApiResponse<T> {
  code: number;
  data: T;
  message: string;
}
const res: ApiResponse<User[]> = { code: 0, data: [], message: 'ok' };

// 泛型类
class Stack<T> {
  private items: T[] = [];
  push(item: T) { this.items.push(item); }
  pop(): T | undefined { return this.items.pop(); }
}
const s = new Stack<number>();
s.push(1);            // 只能 push number
```

### 泛型工具函数

```ts
// 通用请求函数,返回类型由调用方指定
async function request<T>(url: string): Promise<T> {
  const res = await fetch(url);
  return res.json();
}
const users = await request<User[]>('/api/users');   // users: User[]
```

## 高频追问

**Q:泛型是什么?解决什么问题?**
A:泛型是类型的参数化,把类型作为参数传入。解决"既要复用逻辑、又要保留类型安全"的矛盾——不用泛型只能用 any(丢类型)或重复写,泛型让类型在使用时才确定。

**Q:泛型约束(extends)的作用?**
A:限制泛型参数必须满足某种结构,从而能在函数体内安全访问相应属性/方法。如 `T extends { length: number }` 保证 T 有 length。

**Q:`K extends keyof T` 是什么意思?**
A:约束 K 只能是 T 的键名之一。配合 `T[K]` 索引访问类型,实现类型安全地取对象属性,返回值类型精确对应该属性,防止传入不存在的 key。

**Q:泛型和 any 的区别?**
A:any 完全放弃类型检查,输入输出无关联;泛型保留并关联类型信息(输入 T 输出也是 T),在编译期仍做完整检查,类型安全。

**Q:泛型默认值有什么用?**
A:`<T = 默认类型>` 让调用方不显式指定时使用默认类型,提升易用性(类似函数参数默认值),常见于封装组件 props 或工具类型。

## 一句话背诵版

**泛型是类型参数化,让函数/接口/类复用又保留类型安全(优于 any);用 `T extends 约束` 限制结构、`<T = 默认>` 设默认值、`K extends keyof T` + `T[K]` 类型安全取属性;类型在调用时确定,可显式指定或自动推断。**
