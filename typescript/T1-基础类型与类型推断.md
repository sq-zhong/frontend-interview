# T1 · 基础类型与类型推断

> 热度:🔥🔥

## 一句话考点

TS 在 JS 基础上加静态类型;基础类型含 `number/string/boolean/null/undefined/symbol/bigint` + 特殊类型 `any/unknown/never/void`;能自动**类型推断**,不必处处手写类型。

## 原理精讲

### 1. 基础类型

```ts
let n: number = 1;
let s: string = 'a';
let b: boolean = true;
let arr: number[] = [1, 2];
let tuple: [string, number] = ['a', 1];   // 元组:定长定类型
let u: 'success' | 'error' = 'success';    // 字面量联合类型
```

### 2. 四个特殊类型(高频)

| 类型 | 含义 | 用法 |
|------|------|------|
| `any` | 任意类型,**关闭类型检查** | 应尽量避免(失去 TS 意义) |
| `unknown` | 未知类型,**安全的 any** | 用前必须先收窄/断言 |
| `never` | 永不存在的值 | 抛错函数、穷尽检查 |
| `void` | 无返回值 | 函数返回 undefined |

**any vs unknown**:
- any 可赋给任何类型、可任意操作,绕过检查(危险);
- unknown 不能直接操作或赋给别的类型,必须先判断类型收窄,更安全。

### 3. 类型推断

TS 能根据上下文自动推断类型,无需显式标注:

```ts
let x = 3;               // 推断为 number
const y = 3;             // 推断为字面量类型 3(const)
const arr = [1, 'a'];    // 推断 (string | number)[]

function double(n = 1) { // 参数默认值推断 n: number
  return n * 2;          // 返回值推断 number
}
```

### 4. 类型断言

告诉编译器"我确定是某类型"(不做运行时转换):

```ts
const el = document.querySelector('#app') as HTMLDivElement;
const val = someValue as unknown as TargetType;  // 双重断言(慎用)
```

## 代码示例

### unknown 的安全性

```ts
function handle(val: unknown) {
  // val.toFixed();          // ❌ 报错:unknown 不能直接操作
  if (typeof val === 'number') {
    val.toFixed(2);          // ✅ 收窄后可用
  }
}

function handleAny(val: any) {
  val.toFixed();             // ✅ 编译通过但运行时可能崩溃(危险)
}
```

### never 用于穷尽检查

```ts
type Shape = 'circle' | 'square';
function area(shape: Shape) {
  switch (shape) {
    case 'circle': return 1;
    case 'square': return 2;
    default:
      const _exhaustive: never = shape;  // 若新增类型未处理,这里会报错
      return _exhaustive;
  }
}
```

## 高频追问

**Q:any 和 unknown 区别?**
A:any 完全关闭类型检查,可任意赋值和操作,失去 TS 保护;unknown 是"类型安全的 any",不能直接操作,必须先通过类型守卫收窄才能使用。优先用 unknown。

**Q:never 类型有什么用?**
A:表示永不出现的值。用于:① 永远抛错或死循环的函数返回值;② 穷尽性检查(default 分支赋值给 never,漏处理联合类型成员时编译报错);③ 表示不可能的类型交集。

**Q:type 断言 as 会做类型转换吗?**
A:不会。as 只是编译期告诉 TS "把它当作某类型",不产生任何运行时代码,也不做真正的转换。用错会掩盖真实类型 bug。

**Q:什么时候需要手写类型,什么时候靠推断?**
A:函数参数、对象结构、公共 API 边界建议显式标注(可读性 + 契约);局部变量、有初始值的、返回值通常靠推断即可,避免冗余。

**Q:const 和 let 的类型推断有何不同?**
A:let 推断为宽泛类型(如 `let x = 'a'` 推断 string);const 推断为字面量类型(`const x = 'a'` 推断为 `'a'`),因为 const 不可变。

## 一句话背诵版

**TS = JS + 静态类型;基础类型 + 元组/字面量联合;any 关闭检查(危险)、unknown 安全需收窄、never 表示不存在(穷尽检查)、void 无返回;TS 能自动类型推断(const 推字面量、let 推宽泛),as 断言仅编译期不做运行时转换。**
