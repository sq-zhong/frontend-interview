# T8 · 类型体操实战

> 热度:🔥🔥

## 一句话考点

类型体操 = 用映射类型、条件类型、`infer`、递归、模板字面量类型组合出复杂类型;面试常考手写 `DeepPartial`、`Flatten`、模板字面量类型等。

## 原理精讲

综合运用前面的能力(T5 工具类型、T6 映射/keyof、T7 条件类型/infer),核心套路:

1. **遍历键**:映射类型 `[K in keyof T]`;
2. **分支判断**:条件类型 `extends ? :`;
3. **提取类型**:`infer`;
4. **递归**:类型引用自身处理嵌套;
5. **字符串操作**:模板字面量类型 + `Uppercase/Capitalize` 等内置类型。

## 代码示例

### 1. 深度可选 / 深度只读

```ts
type DeepPartial<T> = {
  [K in keyof T]?: T[K] extends object ? DeepPartial<T[K]> : T[K];
};

type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends object ? DeepReadonly<T[K]> : T[K];
};
```

### 2. 模板字面量类型

```ts
// 拼接字符串类型
type Greeting<T extends string> = `Hello, ${T}!`;
type G = Greeting<'World'>;   // 'Hello, World!'

// 事件名转换
type EventName<T extends string> = `on${Capitalize<T>}`;
type E = EventName<'click'>;  // 'onClick'

// 提取路由参数
type ExtractParams<T extends string> =
  T extends `${string}:${infer Param}/${infer Rest}`
    ? Param | ExtractParams<Rest>
    : T extends `${string}:${infer Last}`
      ? Last
      : never;
type Params = ExtractParams<'/user/:id/post/:postId'>;   // 'id' | 'postId'
```

### 3. 元组操作

```ts
// 数组长度
type Length<T extends readonly any[]> = T['length'];
type L = Length<[1, 2, 3]>;   // 3

// 取首/尾元素
type First<T extends any[]> = T extends [infer F, ...any[]] ? F : never;
type Last<T extends any[]> = T extends [...any[], infer L] ? L : never;

// 反转元组
type Reverse<T extends any[]> =
  T extends [infer F, ...infer Rest] ? [...Reverse<Rest>, F] : [];
type R = Reverse<[1, 2, 3]>;   // [3, 2, 1]
```

### 4. 联合转交叉(经典难题)

```ts
type UnionToIntersection<U> =
  (U extends any ? (k: U) => void : never) extends (k: infer I) => void
    ? I : never;
type I = UnionToIntersection<{ a: 1 } | { b: 2 }>;   // { a: 1 } & { b: 2 }
```

### 5. 实用:把接口的部分字段变必填

```ts
// 组合内置工具类型
type RequireFields<T, K extends keyof T> = Omit<T, K> & Required<Pick<T, K>>;

interface Form { name?: string; age?: number; email?: string; }
type ValidForm = RequireFields<Form, 'name' | 'email'>;
// name 和 email 必填,age 仍可选
```

## 高频追问

**Q:实现一个 DeepPartial?**
A:映射类型遍历键 + 条件类型判断值是否为对象,是则递归 DeepPartial,否则保留:`{ [K in keyof T]?: T[K] extends object ? DeepPartial<T[K]> : T[K] }`。

**Q:模板字面量类型能做什么?**
A:在类型层面拼接/解析字符串。如生成事件名 `on${Capitalize<T>}`、约束字符串格式、从路由字符串提取参数名(配合 infer 递归)。有 Uppercase/Lowercase/Capitalize/Uncapitalize 四个内置字符串类型。

**Q:如何取元组的长度/首尾元素?**
A:长度用 `T['length']`;首元素 `T extends [infer F, ...any[]] ? F : never`;尾元素 `T extends [...any[], infer L] ? L : never`。用 infer + 剩余元素模式匹配。

**Q:类型体操在实际开发有用吗?**
A:大多数业务用内置工具类型就够。复杂类型体操主要用于:封装类型安全的库/工具、精确的 API 类型推导、表单/路由/状态的类型约束。理解它有助于读懂库的类型定义和排查类型报错。

**Q:递归类型有什么限制?**
A:TS 对递归深度有限制(约 50 层,过深报"类型实例化过深");尾递归优化有限。深层结构或大元组要注意性能和深度限制。

## 一句话背诵版

**类型体操 = 映射类型(遍历键)+ 条件类型(分支)+ infer(提取)+ 递归(嵌套)+ 模板字面量(字符串操作);常考 DeepPartial/DeepReadonly、元组 First/Last/Reverse、模板字面量提参、联合转交叉;业务多用内置工具类型,复杂体操用于库和精确类型约束。**
