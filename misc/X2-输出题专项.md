# X2 · 输出题专项

> 热度:🔥🔥

综合考察 **事件循环 + this + 变量提升 + 闭包 + 类型转换**。做题口诀:**先同步、再微任务、后宏任务;this 看调用;var 提升、let 死区。**

## 一、事件循环输出题

### 题 1(async/await + Promise + setTimeout)

```js
console.log('start');
setTimeout(() => console.log('timeout'), 0);
Promise.resolve().then(() => console.log('promise1'))
  .then(() => console.log('promise2'));
(async () => {
  console.log('async start');
  await null;
  console.log('async end');
})();
console.log('end');
```
**输出**:
```
start
async start
end
promise1
async end     // await 后是微任务,注册时机在 promise1 之后
promise2
timeout
```
**解析**:同步(start / async start / end)→ 微任务队列按注册顺序(promise1 先注册,async end 的微任务在 await 时注册,晚于 promise1)→ promise2 → 宏任务 timeout。

### 题 2(嵌套定时器与微任务)

```js
setTimeout(() => {
  console.log('t1');
  Promise.resolve().then(() => console.log('p1'));
}, 0);
setTimeout(() => {
  console.log('t2');
  Promise.resolve().then(() => console.log('p2'));
}, 0);
```
**输出**:`t1 → p1 → t2 → p2`
**解析**:每执行完一个宏任务(t1)就清空微任务(p1),再执行下一个宏任务(t2)→ 清空(p2)。

## 二、this 输出题

```js
const obj = {
  count: 0,
  inc: function () {
    setTimeout(function () { this.count++; console.log(this.count); }, 0);
  },
  incArrow: function () {
    setTimeout(() => { this.count++; console.log(this.count); }, 0);
  },
};
obj.inc();       // NaN(普通函数 this 指向 window,window.count 是 undefined)
obj.incArrow();  // 1(箭头函数 this 指向 obj)
```

## 三、变量提升与作用域

### 题 3(var 循环经典)

```js
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}
// 输出:3 3 3(var 无块级作用域,三个回调共享同一个 i,循环结束 i=3)
// 改 let → 0 1 2
```

### 题 4(函数提升 vs 变量提升)

```js
console.log(a);      // [Function a](函数声明整体提升,优先级高于 var)
var a = 1;
function a() {}
console.log(a);      // 1(赋值后)
```

### 题 5(暂时性死区)

```js
console.log(x);   // ReferenceError(let 存在 TDZ,声明前不可访问)
let x = 1;
```

## 四、类型转换

```js
[] + [];        // ''(两个空数组转字符串拼接)
[] + {};        // '[object Object]'
{} + [];        // 0(块级 {} 被当语句,+[] 转数字 0)—— 在控制台里
1 + '1';        // '11'(数字转字符串)
'1' - 1;        // 0('1' 转数字)
true + 1;       // 2(true 转 1)
null + 1;       // 1(null 转 0)
undefined + 1;  // NaN
[1,2] == '1,2'; // true(数组转字符串比较)
```

## 五、综合题(this + 闭包 + 异步)

```js
var name = 'global';
const person = {
  name: 'person',
  say() {
    return function () { return this.name; };   // 返回普通函数
  },
  sayArrow() {
    return () => this.name;                     // 返回箭头函数
  },
};
console.log(person.say()());       // 'global'(普通函数独立调用,this=window)
console.log(person.sayArrow()());  // 'person'(箭头函数继承 sayArrow 的 this)
```

## 高频追问

**Q:做输出题的思路?**
A:① 先扫同步代码从上到下;② 遇到异步分类:Promise.then/await 入微任务,setTimeout 入宏任务;③ 同步完清空所有微任务;④ 取一个宏任务再清空微任务,循环;⑤ this 单独看调用方式,箭头看外层。

**Q:`await` 后的代码在哪个时机?**
A:相当于 `.then` 回调,作为微任务,在当前同步代码执行完后、按注册顺序执行。

## 一句话背诵版

**口诀:同步 → 清空微任务(Promise/await)→ 一个宏任务(setTimeout)→ 再清微任务;this 看调用方式、箭头看外层;var 提升(函数声明优先)、let 有 TDZ;循环共享变量用 let 解决。**
